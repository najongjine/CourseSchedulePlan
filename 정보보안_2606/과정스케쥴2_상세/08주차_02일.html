한 주문에 여러 상품 저장하기

주문 머리 한 건을 만들고 그 주문에 연결된 상품 행들을 입력한다.

8주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 주문 데이터 입력 순서를 이해한다.
  • 외래키 값을 맞춰 INSERT한다.
  • 저장된 원시 행을 조회한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

먼저 영수증 번호를 만들고, 그 번호를 상품 줄마다 적는다

order_items를 먼저 입력하려면 어느 주문인지 알 수 없다. 따라서 orders 행을 먼저 만들고 생성된 order_id를 확인한다.

수업에서는 복잡한 자동 반환보다 SELECT로 번호를 확인한 뒤 직접 상세 행에 입력한다.

orders INSERT민지→SELECTorder_id=1→items INSERT1, 상품1→items INSERT1, 상품2

손가락 실습: 주문 번호 스티커를 먼저 만들고 두 상품 카드에 같은 번호를 붙인다.

직접 따라치기

상품 ID를 조회한 뒤 주문 한 건과 상세 두 건을 입력한다.

코드를 입력한 뒤 반드시 실행한다

INSERT INTO orders (customer_name) VALUES ('민지');

SELECT order_id, customer_name FROM orders;
SELECT product_id, name, price FROM products;

INSERT INTO order_items (order_id, product_id, quantity, unit_price)
VALUES (1, 1, 2, 1200);

INSERT INTO order_items (order_id, product_id, quantity, unit_price)
VALUES (1, 2, 1, 800);

SELECT order_id, product_id, quantity, unit_price
FROM order_items;

실행 결과 예

order_id | product_id | quantity | unit_price ----------+------------+----------+----------- 1 | 1 | 2 | 1200 1 | 2 | 1 | 800

한 줄씩 읽어 보기

  1. orders 행이 먼저 있어야 order_items의 order_id=1 연결이 유효하다.
  2. 상품 번호 역시 products에 먼저 있어야 한다.
  3. 같은 order_id 두 줄이 하나의 장바구니 주문을 나타낸다.
입력한 뒤 바꾸어 볼 값

지훈의 주문 한 건에 초콜릿 3개를 입력하는 SQL을 작성한다. 실제 order_id는 조회 후 맞춘다.

손으로 추적하는 문제

외래키가 있는 입력은 부모 행이 먼저, 연결되는 자식 행이 다음이라는 순서를 익힌다.

-- orders에 order_id 2가 이미 존재하고 products의 3번 가격은 1500이다.
INSERT INTO order_items (order_id, product_id, quantity, unit_price)
VALUES (2, 3, 2, 1500);

문제. 이 행이 나타내는 구매 금액은 얼마인가?

정답과 풀이 열기

상품 3번을 단가 1500에 2개 구매한 상세 행이므로 금액은 3000원이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 주문 세 건을 직접 입력한다.
  2. 주문 하나에는 상품 두 종류 이상을 넣어 본다.
  3. 없는 product_id를 쓰면 왜 거부되어야 하는지 적는다.
연습 확인 포인트

이제 데이터는 들어갔지만 상품 이름과 주문자를 한 표로 보려면 JOIN이 필요하다.
