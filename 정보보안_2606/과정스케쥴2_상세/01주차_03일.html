조건문으로 UP과 DOWN 가르기

같은 입력이라도 정답보다 작거나 큰지에 따라 다른 문장을 보여 주는 게임의 핵심 날이다.

1주차 3일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 비교식의 참과 거짓을 이해한다.
  • `if / else if / else`를 작성한다.
  • 조건 순서가 결과에 미치는 영향을 추적한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

조건문은 세 갈래 표지판

프로그램은 모든 길을 동시에 걷지 않는다. 위에서부터 조건을 검사하여 처음 참인 길 하나만 실행한다.

정답보다 작은지, 큰지, 둘 다 아닌지를 순서대로 묻으면 마지막 길은 자연스럽게 정답인 경우다.

guess23→guess < answer?참→선택된 길UP 출력

손가락 실습: 정답을 30으로 정하고 교사가 23, 45, 30 카드를 들 때 어느 길로 갈지 손으로 표시한다.

직접 따라치기

정답은 우선 고정값 30으로 둔다. 세 입력을 각각 실행해 결과가 어떻게 갈리는지 확인한다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int main(void) {
    int answer = 30;
    int guess;

    printf("숫자를 입력하세요: ");
    scanf("%d", &guess);

    if (guess < answer) {
        printf("UP\n");
    } else if (guess > answer) {
        printf("DOWN\n");
    } else {
        printf("정답!\n");
    }
    return 0;
}

실행 결과 예

숫자를 입력하세요: 23 UP

한 줄씩 읽어 보기

  1. `guess < answer`는 23 < 30이므로 참이다.
  2. 첫 조건이 참이면 아래의 `else if`, `else`는 실행하지 않는다.
  3. 30을 입력하면 두 비교가 모두 거짓이므로 마지막 `else`가 실행된다.
입력한 뒤 바꾸어 볼 값

`answer`를 50으로 변경하고 입력 50, 60, 40의 문구를 실행 전 각각 적는다.

손으로 추적하는 문제

기사 문제에서는 조건의 경계값, 특히 `<`와 `<=`의 차이를 노린다.

int score = 60;
if (score > 60) {
    printf("A");
} else if (score >= 60) {
    printf("B");
} else {
    printf("C");
}

문제. 출력 문자는 무엇이며, 첫 조건이 거짓인 이유는 무엇인가?

정답과 풀이 열기

`score > 60`은 60이 60보다 크지 않으므로 거짓이다. 두 번째 `score >= 60`은 참이므로 `B`가 출력된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 정답이 10인 홀짝 아닌 숫자 맞히기 판정 코드를 직접 작성한다.
  2. 입력 가능 범위가 1~100이 아니면 `범위 오류`를 먼저 출력하도록 조건을 추가한다.
  3. `<` 하나를 `<=`로 바꿨을 때 정답 입력에서 발생하는 문제를 설명한다.
연습 확인 포인트

정답까지 UP으로 처리되는 버그가 생길 수 있다. 경계값을 직접 입력해 보는 습관이 중요하다.

금요일 작품으로 연결

금요일: 업다운 숫자 맞히기 게임

입력 한 번에 대해 UP, DOWN, 정답을 출력하는 게임의 판정 엔진을 완성한다.
