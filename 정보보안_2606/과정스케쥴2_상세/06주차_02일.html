장바구니에 소계를 쌓기

여러 상품의 이름과 수량을 입력하여 각 소계와 전체 총액을 계산한다.

6주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 반복해서 상품을 입력한다.
  • 가격과 수량을 곱한다.
  • 총액 누적을 추적한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

장바구니 총액은 고른 상품의 소계가 차곡차곡 더해진 값

상품 한 건의 가격 곱하기 수량은 소계이고, 여러 소계를 더한 값이 할인 전 총액이다.

아직 데이터베이스는 없다. 입력하는 동안 메모리의 `total` 상자가 장바구니 합계를 기억한다.

물 x21600→초콜릿 x11500→total3100→할인 전3100

손가락 실습: 상품과 수량 카드로 소계를 계산하고, 계산대 total 판에 차례로 더한다.

직접 따라치기

구매 종류를 두 개로 제한하여 반복과 누적을 명확히 익힌다.

코드를 입력한 뒤 반드시 실행한다

products = {"콜라": 1200, "물": 800, "초콜릿": 1500}
total = 0

for i in range(2):
    name = input("상품명: ")
    count = int(input("수량: "))
    if name in products and count > 0:
        subtotal = products[name] * count
        total += subtotal
        print("소계:", subtotal)
    else:
        print("입력을 확인하세요.")

print("할인 전 총액:", total)

실행 결과 예

상품명: 물 수량: 2 소계: 1600 상품명: 초콜릿 수량: 1 소계: 1500 할인 전 총액: 3100

한 줄씩 읽어 보기

  1. `total`은 반복 전 0으로 만들어 두 상품의 소계를 유지한다.
  2. 정상 입력인 경우에만 subtotal을 계산하고 total에 더한다.
  3. 잘못된 상품은 총액에 포함되지 않는다.
입력한 뒤 바꾸어 볼 값

두 번째 상품을 없는 이름으로 입력하면 총액이 첫 소계만 남는 이유를 확인한다.

손으로 추적하는 문제

리스트가 없어도 누적 변수는 기사 코드 추적의 중요한 기본이다.

items = {"pen": 500, "book": 2000}
total = 0
for name in ["pen", "book", "pen"]:
    total += items[name]
print(total)

문제. 출력 총액은 무엇인가?

정답과 풀이 열기

펜 500, 책 2000, 펜 500을 더해 `3000`이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 상품 종류 수를 3으로 늘리고 total 변화를 표로 쓴다.
  2. 수량 0이나 음수를 입력했을 때 총액이 늘지 않게 확인한다.
  3. 소계와 총액의 차이를 한 문장으로 설명한다.
연습 확인 포인트

할인은 총액이 완성된 다음 적용한다. 계산 순서를 분리하면 실수가 적다.
