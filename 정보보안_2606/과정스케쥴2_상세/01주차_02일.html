키보드 입력을 숫자로 받기

사용자가 숫자를 입력해야 게임이 대화하듯 반응한다. 입력이 변수에 들어가는 과정을 확인한다.

1주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • `scanf`로 정수를 입력받는다.
  • 입력값으로 계산한 결과를 출력한다.
  • 입력-저장-처리-출력 흐름을 말할 수 있다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

입력은 키보드에서 변수 상자로 이동하는 배달

화면에 7을 타이핑하는 것만으로 프로그램이 그 숫자를 아는 것은 아니다. `scanf`가 숫자를 읽어 `guess` 주소의 상자에 넣는다.

`&guess`의 `&`는 오늘은 'guess 상자의 위치'라고 이해한다. 깊은 포인터 설명은 나중에 한다.

키보드7 입력→scanf&guess→상자guess: 7→printf입력: 7

손가락 실습: 한 학생이 숫자 카드를 들고, 다른 학생이 `guess` 이름표 상자 역할을 해 카드를 전달해 본다.

직접 따라치기

값 두 개를 입력하고 계산해 보자. 컴파일 후 8과 3을 차례대로 입력한다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int main(void) {
    int first;
    int second;

    printf("첫 숫자: ");
    scanf("%d", &first);
    printf("둘째 숫자: ");
    scanf("%d", &second);

    printf("합계: %d\n", first + second);
    printf("차이: %d\n", first - second);
    return 0;
}

실행 결과 예

첫 숫자: 8 둘째 숫자: 3 합계: 11 차이: 5

한 줄씩 읽어 보기

  1. `first`, `second`는 선언 직후 아직 수업에서 믿고 쓸 값이 없는 빈 상자다.
  2. 첫 번째 `scanf`가 끝나면 `first`는 8이다.
  3. 출력할 때는 상자를 바꾸지 않고 두 값을 읽어 계산한다.
입력한 뒤 바꾸어 볼 값

입력을 3과 8로 바꾸면 차이가 음수가 된다. 실행하기 전에 결과를 적어 본다.

손으로 추적하는 문제

입력문이 있어도 시험 문제에서는 입력값이 주어지고, 그 뒤 계산을 손으로 수행하면 된다.

int n = 4;
int result = n * 3;
result = result - n;
printf("%d", result);

문제. `result`의 최종 출력은 무엇인가?

정답과 풀이 열기

먼저 4 * 3으로 12, 그 다음 12 - 4로 8이 된다. 출력은 `8`이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 나이와 내년 나이를 입력/계산하는 프로그램을 작성한다.
  2. `price`와 `count`를 입력받아 결제 금액을 출력한다.
  3. 게임용 `guess`를 입력받아 `당신의 선택: 숫자`를 출력한다.
연습 확인 포인트

곱셈은 금액 계산으로, `guess` 입력은 바로 금요일 게임의 한 조각으로 다시 사용된다.

금요일 작품으로 연결

금요일: 업다운 숫자 맞히기 게임

게임의 사용자가 추측한 숫자를 입력받는 부분을 완성한다. 아직 정답 비교는 하지 않는다.
