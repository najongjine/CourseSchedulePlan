컴퓨터에게 값을 맡기는 첫날

게임을 만들기 전에 컴퓨터가 문장을 출력하고 숫자를 기억하는 방식을 몸으로 익힌다.

  • 소스 코드와 실행 결과를 구분한다.
  • 정수 변수에 값을 저장하고 출력한다.

4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

변수는 이름표가 붙은 한 칸짜리 상자

컴퓨터는 의미를 짐작하지 않는다. `score`라는 상자를 만들고 10을 넣으라고 적어야 10을 기억한다.

`score = score + 5`는 수학 등식이 아니라, 기존 상자의 값을 꺼내 5를 더한 뒤 다시 넣으라는 명령이다.

코드score = 10→메모리 상자score: 10→출력10점

손가락 실습: 종이에 score 상자를 그리고 10, 15를 차례대로 지우고 다시 써 본다.

직접 따라치기

아래 코드를 새 파일 `day1.c`에 한 줄씩 입력한다. 세미콜론과 큰따옴표도 코드의 일부다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int main(void) {
    int score = 10;
    printf("시작 점수: %d\n", score);

    score = score + 5;
    printf("문제를 맞힌 뒤: %d\n", score);
    return 0;
}

실행 결과 예

시작 점수: 10 문제를 맞힌 뒤: 15

한 줄씩 읽어 보기

  1. `int score = 10;`에서 정수를 담을 상자를 만들고 10을 넣는다.
  2. `%d` 위치에 현재 `score`의 정수 값이 들어간다.
  3. `score = score + 5;` 실행 뒤 상자에는 15가 남는다.
입력한 뒤 바꾸어 볼 값

`score`의 시작값을 20으로, 더하는 값을 3으로 바꾸고 실행 전 출력 두 줄을 먼저 예측한다.

손으로 추적하는 문제

정보처리기사 코드 문제는 실행 뒤 변수에 남은 값을 묻는 일이 많다. 종이에 상자를 그려 추적한다.

int a = 3;
int b = a + 2;
a = b * 2;
printf("%d %d", a, b);

문제. 출력되는 두 숫자는 무엇인가?

정답과 풀이 열기

`b`에는 5가 저장되고, 그 뒤 `a`에는 5 * 2인 10이 저장된다. 출력은 `10 5`이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. `coin` 변수를 만들고 500을 저장하여 `보유 코인: 500`을 출력한다.
  2. `life`를 3으로 시작하여 1을 뺀 뒤 두 번 출력되는 값을 예상하고 실행한다.
  3. `%d`를 빠뜨리거나 세미콜론을 지우고 컴파일 오류 메시지를 관찰한 뒤 되돌린다.
연습 확인 포인트

중요한 것은 정답보다 실행 순서다. 값이 바뀔 때마다 종이 상자의 숫자를 갱신하면 된다.
