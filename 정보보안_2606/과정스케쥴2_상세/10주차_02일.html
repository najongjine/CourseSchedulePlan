fetch로 DB 상품 카드 표시하기

지난주 파일 안 배열 대신 API에서 받은 상품 배열을 상태에 저장하여 같은 카드 화면을 그린다.

10주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결

오늘의 목표

  • `useState`의 저장 역할을 안다.
  • `useEffect`에서 한 번 조회한다.
  • API 결과가 map으로 표시됨을 확인한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

상태는 화면이 기억하고 다시 그릴 현재 데이터

샘플 배열은 고정된 글씨였지만 API 상품은 요청이 끝난 뒤 도착한다. React는 도착한 배열을 `products` 상태에 저장하고 새 화면을 그린다.

처음 빈 배열이라 카드가 없다가, fetch 완료 뒤 DB 상품 카드가 나타난다.

처음[]→fetchGET /products→setProductsDB 배열→render카드 표시

손가락 실습: 빈 진열대를 그린 뒤 배송 트럭(API 결과)이 도착하면 카드 네 장을 올리는 장면을 그린다.

직접 따라치기

기존 ProductCard는 유지하고 App에서 상품을 가져오는 부분을 작성한다.

코드를 입력한 뒤 반드시 실행한다

import { useEffect, useState } from "react";

export default function App() {
  const [products, setProducts] = useState([]);

  useEffect(() => {
    fetch("http://localhost:8000/products")
      .then(response => response.json())
      .then(data => setProducts(data));
  }, []);

  return (
    <main>
      <h1>DB 상품 목록</h1>
      {products.map(product => (
        <ProductCard key={product.product_id} {...product} />
      ))}
    </main>
  );
}

실행 결과 예

PostgreSQL products에 저장된 상품들이 React 카드로 표시됨

한 줄씩 읽어 보기

  1. 상태의 시작값은 빈 배열이다.
  2. `[]`가 있는 useEffect는 화면 시작 시 한 번 상품을 요청하는 구조다.
  3. 응답 데이터가 상태에 들어가면 map이 카드들을 만든다.
입력한 뒤 바꾸어 볼 값

DB에서 상품 가격 하나를 UPDATE하고 브라우저를 새로고침해 카드 가격 변화가 반영되는지 확인한다.

손으로 추적하는 문제

조회 화면의 정확성은 결국 서버가 실행한 SQL 결과가 맞는지에서 출발한다.

-- DB 가격을 변경한 후 API가 실행하는 문장
SELECT name, price
FROM products
WHERE product_id = 2;

문제. 상품 2 가격을 1800으로 UPDATE했다면 화면에는 어떤 가격이 보여야 하는가?

정답과 풀이 열기

API가 최신 행을 다시 조회하므로 상품 2 카드는 1800원을 표시해야 한다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. API가 꺼져 있을 때 화면이 데이터를 못 받는 상황을 관찰한다.
  2. DB 상품 하나를 변경하고 화면 결과를 확인한다.
  3. 샘플 배열과 API 상태 배열의 차이를 적는다.
연습 확인 포인트
