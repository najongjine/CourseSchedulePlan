먼저 감각으로 이해하기

게임은 입력-판정-피드백의 반복

게임이 재미있는 이유는 사용자의 입력이 바로 다음 반응을 만든다는 데 있다. 오늘의 코드는 화면은 단순하지만 실제 게임의 기본 고리를 갖는다.

코드를 외우는 것이 목적이 아니다. `guess` 상자와 `count` 상자가 매 시도마다 어떻게 달라지는지 보이면 성공이다.

입력guess→판정UP/DOWN→기록count→종료정답

손가락 실습: 친구가 입력을 말하면 기록 담당이 표에 guess와 count를 적고 실행 화면과 대조한다.

직접 따라치기

먼저 필수 기능을 그대로 입력해 성공시킨 뒤, 시간이 남는 학생만 범위 오류 검사를 추가한다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int main(void) {
    int answer = 30;
    int guess = 0;
    int count = 0;

    printf("=== UP & DOWN ===\n");
    while (guess != answer) {
        printf("1~100 숫자 입력: ");
        scanf("%d", &guess);

        if (guess < 1 || guess > 100) {
            printf("범위를 벗어났습니다.\n");
            continue;
        }

        count++;
        if (guess < answer) {
            printf("UP\n");
        } else if (guess > answer) {
            printf("DOWN\n");
        } else {
            printf("정답!\n");
        }
    }
    printf("시도 횟수: %d\n", count);
    return 0;
}

실행 결과 예

=== UP & DOWN === 1~100 숫자 입력: 101 범위를 벗어났습니다. 1~100 숫자 입력: 20 UP 1~100 숫자 입력: 30 정답! 시도 횟수: 2

한 줄씩 읽어 보기

  1. 범위 오류 입력은 `continue`로 아래 판정과 count 증가를 건너뛴다.
  2. `count++`는 올바른 범위의 추측만 센다는 게임 규칙이다.
  3. 맞힌 뒤 `guess != answer`가 거짓이 되면서 반복이 종료된다.
입력한 뒤 바꾸어 볼 값

범위 오류도 횟수에 포함하는 게임으로 규칙을 바꾸려면 `count++`를 어디로 옮겨야 하는가?

손으로 추적하는 문제

코드 추적에서는 `continue`가 아래 문장을 실행하지 않고 곧장 조건 검사로 돌아간다는 점을 놓치지 않는다.

int n = 0;
int count = 0;
while (n < 5) {
    n++;
    if (n == 3) continue;
    count++;
}
printf("%d", count);

문제. `count`의 출력값은 무엇인가?

정답과 풀이 열기

n은 1,2,3,4,5가 된다. n이 3인 바퀴만 `count++`를 건너뛰므로 총 네 번 증가하여 `4`가 출력된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 정답 30에 대해 `101, 20, 30`을 입력하고 예상 count와 실제 count를 비교한다.
  2. 정답을 77로 바꿔 친구가 플레이하게 하고 UP/DOWN 문구가 맞는지 검사한다.
  3. 프로그램의 입력, 처리, 출력, 반복을 네 문장으로 적는다.
