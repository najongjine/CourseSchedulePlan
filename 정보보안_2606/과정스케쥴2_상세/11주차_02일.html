장바구니 총액 계산하기

장바구니 배열에 담긴 상품 가격을 합산해 주문 전 총액을 보여 준다.

오늘의 목표

  • `reduce` 누적 계산을 이해한다.
  • 빈 장바구니 총액을 처리한다.
  • Python 할인 계산과 연결한다.

먼저 감각으로 이해하기

reduce는 장바구니 항목을 계산대 위로 하나씩 올리는 일

Python에서는 반복문으로 `total += price`를 썼다. JavaScript의 `reduce`는 같은 합산을 짧게 표현한다.

필수 기능은 상품을 여러 번 담으면 각각 한 개씩 계산하는 단순 규칙이다.

시작 sum0→노트+2000→펜+1000→총액3000

손가락 실습: 6주차 Python total 변화표 옆에 React cart sum 변화표를 동일하게 작성한다.

직접 따라치기

장바구니 표시 아래 총액 한 줄을 추가한다.

코드를 입력한 뒤 반드시 실행한다

const total = cart.reduce((sum, item) => {
  return sum + item.price;
}, 0);

return (
  <aside>
    <h2>장바구니</h2>
    {cart.map((item, index) => (
      <p key={index}>{item.name} {item.price}원</p>
    ))}
    <strong>총액: {total.toLocaleString()}원</strong>
  </aside>
);

실행 결과 예

장바구니: 노트 2000원 / 펜 1000원 총액: 3,000원

한 줄씩 읽어 보기

  1. reduce의 sum은 0에서 시작한다.
  2. 각 item의 price가 sum에 추가되어 최종 숫자가 total이 된다.
  3. cart가 비어 있으면 반복할 항목이 없어 total은 초기값 0이다.
입력한 뒤 바꾸어 볼 값

동일한 노트를 두 번 담았을 때 총액이 두 배가 되는지 먼저 예상하고 확인한다.

손으로 추적하는 문제

언어가 달라도 누적 합계를 추적하는 표는 똑같이 사용할 수 있다.

const prices = [1200, 800, 1500];
const total = prices.reduce((sum, price) => sum + price, 0);
const discount = total >= 3000 ? 500 : 0;

문제. `total`과 `discount`는 무엇인가?

정답과 풀이 열기

합계는 3500이고 3000 이상 조건을 만족하므로 discount는 500이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 빈 장바구니에서 총액 0을 확인한다.
  2. 상품 세 개를 담아 손계산과 화면 총액을 비교한다.
  3. 6주차 할인 계산을 선택 기능으로 화면에 추가할 위치를 표시한다.
