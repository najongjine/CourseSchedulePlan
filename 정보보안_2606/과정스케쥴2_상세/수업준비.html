학생은 도구 설치보다 코드를 실행해 결과가 바뀌는 경험에 집중한다. 아래 환경은 강사가 수업 전에 준비하고, 수업 첫 시간에는 실행 확인만 함께 한다.

수업 범위와 설치 원칙

  • 1~2주차 C 코드는 표준 헤더 stdio.h만 사용한다. 별도 C 라이브러리는 설치하지 않는다.
  • 3~4주차 Java 코드는 JDK 표준 클래스인 java.util.Scanner만 사용한다. 외부 JAR는 없다.
  • 5~6주차 Python 코드는 내장 문법과 내장 함수만 사용한다. pip install은 하지 않는다.
  • 7~8주차는 PostgreSQL과 SQL 실행 도구를 강사가 미리 준비한다. SQL은 표준 문장을 우선 사용한다.
  • 9~12주차는 강사가 React 기본 프로젝트와 Python API 연결 뼈대를 제공한다. 학생은 매 차시 학습 기능만 직접 작성한다.

언어별 실행 확인

학생 자리에 앉아 아래 명령이 실행되는지만 수업 전에 확인한다. 경로 또는 설치 문제는 학생 실습 시간 전에 해결한다.

C언어

gcc --version
gcc day1.c -o day1
./day1

Java

javac -version
javac WalletApp.java
java WalletApp

Python

python --version
python study_report.py

PostgreSQL

psql --version
psql -U 수업계정 -d shop_class

폴더를 미리 나누는 이유

노베이스 학생은 실행 파일, 소스 파일, 데이터 파일을 섞어 놓으면 오류 원인을 찾기 어렵다. 주차별 폴더를 미리 제공하면 학생은 오늘 작성할 파일 위치부터 확신할 수 있다.

PostgreSQL 수업 DB 기준

  • 학생마다 자신의 연습 DB를 사용하거나, 삭제/수정 실습이 가능한 별도 수업 DB를 제공한다.
  • 기본키 자동 생성은 INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY를 우선 사용한다.
  • SERIAL, RETURNING, ILIKE를 사용할 때에는 PostgreSQL 확장 문법임을 반드시 표시한다.
  • UPDATE와 DELETE 실습은 항상 같은 WHERE의 확인용 SELECT를 먼저 실행한다.

강사가 준비해 둘 공통 뼈대

C, Java, Python: 빈 주차 폴더와 실행 방법 메모
SQL: 접속 가능한 PostgreSQL DB와 초기화 스크립트
React: 실행 확인된 기본 앱과 CSS 파일
API: DB 접속정보를 제외한 조회/등록/주문 서버 뼈대

React/API 설치와 서버 구성 자체는 이 과정의 기사 대비 핵심이 아니므로 강사가 사전에 안정화한다. 학생은 화면 코드, 요청 흐름, SQL이 만나는 지점을 직접 작성하고 설명한다.
