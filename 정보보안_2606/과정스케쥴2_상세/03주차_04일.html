메뉴를 반복하고 지출 메서드 붙이기

지갑 객체에 지출 규칙을 넣고 사용자가 종료할 때까지 메뉴를 반복한다.

3주차 4일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 지출 메서드가 필드를 안전하게 바꾼다.
  • `while` 메뉴를 구성한다.
  • 객체 메서드 호출 순서를 추적한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

객체가 스스로 규칙을 지키게 한다

지갑 밖에서 잔액을 마구 빼기보다 `spend` 메서드가 돈이 충분한지 확인한 뒤 변경하게 만든다.

사용자는 메뉴 번호를 선택하고, 반복문은 종료 번호를 선택할 때까지 다시 메뉴를 보여 준다.

메뉴 선택2 지출→spend잔액 검사→balance감소/유지→다음 메뉴반복

손가락 실습: 메뉴 1=입금, 2=지출, 3=확인, 0=종료 카드를 순서대로 들며 지갑 상자를 갱신한다.

직접 따라치기

금요일 완성본 직전 코드다. `spend`의 조건과 반복 종료를 유심히 본다.

코드를 입력한 뒤 반드시 실행한다

class Wallet {
    int balance;
    void deposit(int amount) { balance += amount; }
    void spend(int amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("사용 완료");
        } else {
            System.out.println("사용할 수 없습니다.");
        }
    }
}

public class WalletMenu {
    public static void main(String[] args) {
        Wallet wallet = new Wallet();
        wallet.deposit(10000);
        wallet.spend(3000);
        wallet.spend(9000);
        System.out.println(wallet.balance);
    }
}

실행 결과 예

사용 완료 사용할 수 없습니다. 7000

한 줄씩 읽어 보기

  1. 처음 입금 후 balance는 10000이다.
  2. 3000 지출은 성공하여 7000으로 줄어든다.
  3. 9000 지출은 현재 잔액보다 크므로 거절되고 7000이 유지된다.
입력한 뒤 바꾸어 볼 값

입금을 5000, 지출을 5000으로 바꿨을 때 잔액과 메시지를 예측한다.

손으로 추적하는 문제

메서드 호출이 여러 번이면 호출 직후마다 객체 필드를 한 행씩 기록한다.

class Account {
    int money = 100;
    void use(int n) {
        if (money >= n) money -= n;
        else money += 10;
    }
}
Account a = new Account();
a.use(60);
a.use(50);
System.out.print(a.money);

문제. 최종 `money`의 값은 무엇인가?

정답과 풀이 열기

첫 사용 뒤 40이 남는다. 두 번째 50은 부족하므로 10을 더하여 50이 된다. 출력은 `50`.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. Scanner 메뉴 반복의 의사코드를 `선택 입력 -> 메서드 실행 -> 다시 메뉴` 순서로 적는다.
  2. 지출 0 또는 음수를 거부하는 조건을 시험한다.
  3. 필드값이 바뀌는 메서드 줄에 형광펜 표시를 한다.
연습 확인 포인트

객체 코드를 읽을 때 모든 줄보다 필드를 바꾸는 줄을 먼저 찾으면 흐름이 빠르게 보인다.
