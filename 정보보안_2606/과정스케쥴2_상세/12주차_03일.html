먼저 감각으로 이해하기

화면 총액과 DB 매출은 같은 주문을 다른 위치에서 계산한 값

화면의 total은 주문 전 사용자에게 보여 주는 값이고, DB의 SUM은 저장 뒤 기록에서 다시 계산한 값이다.

두 값이 다르면 주문 데이터나 계산 로직에 오류가 있다는 신호다.

cart total화면→POST저장→order_items기록→SUM검증

손가락 실습: 같은 주문에 대해 화면 계산표와 SQL 계산표를 좌우로 쓰고 숫자가 일치하는지 선으로 연결한다.

직접 따라치기

Python 추적 문제를 입력하며 장바구니 계산의 원형을 되새긴다.

코드를 입력한 뒤 반드시 실행한다

def cart_total(items):
    total = 0
    for item in items:
        total += item["price"] * item["count"]
    if total >= 5000:
        total -= 500
    return total

cart = [
    {"price": 1200, "count": 2},
    {"price": 1500, "count": 2}
]
print(cart_total(cart))

실행 결과 예

4900

한 줄씩 읽어 보기

  1. 첫 상품 소계는 2400, 둘째는 3000으로 할인 전 합계는 5400이다.
  2. 5000 이상이므로 500을 빼 최종 4900이다.
  3. 이전 Python 프로젝트의 할인 계산과 같은 형태다.
입력한 뒤 바꾸어 볼 값
