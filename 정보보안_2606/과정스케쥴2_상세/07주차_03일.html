WHERE와 ORDER BY로 원하는 상품 찾기

가격과 재고 조건으로 필요한 상품만 조회하고 정렬한다.

7주차 3일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • WHERE 조건을 작성한다.
  • 정렬 방향을 구분한다.
  • 조회는 저장값을 바꾸지 않음을 안다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

조회 조건은 진열대에서 필요한 카드만 꺼내 보는 필터

`WHERE price >= 1000`은 비싼 상품 카드만 통과시키는 체다.

`ORDER BY price DESC`는 통과한 카드의 줄 세우기 순서를 높은 가격부터로 바꾼다. DB 안 원본 위치를 바꾸는 것은 아니다.

전체 상품10건→WHERE가격 >=1000→ORDER BY비싼 순→결과선별 표

손가락 실습: 상품 카드들을 가격 조건 선 뒤로 통과시키고 통과한 카드만 비싼 순으로 늘어놓는다.

직접 따라치기

가격과 재고에 대한 자주 쓰는 조회를 직접 실행한다.

코드를 입력한 뒤 반드시 실행한다

SELECT name, price, stock
FROM products
WHERE price >= 1000
ORDER BY price DESC;

SELECT name, stock
FROM products
WHERE stock < 10
ORDER BY stock ASC;

실행 결과 예

-- 첫 조회: 1000원 이상 상품을 비싼 순으로 표시 -- 둘째 조회: 재고 10개 미만 상품을 적은 순으로 표시

한 줄씩 읽어 보기

  1. FROM에서 대상 표를 정하고 WHERE에서 남길 행을 고른다.
  2. ORDER BY는 결과 행 순서를 정한다.
  3. `DESC`는 큰 값부터, `ASC`는 작은 값부터다.
입력한 뒤 바꾸어 볼 값

조건을 `price BETWEEN 800 AND 1500`으로 작성해 포함되는 상품을 예상한다.

손으로 추적하는 문제

기사에서는 WHERE 조건을 만족하는 행 수와 ORDER BY 결과 순서를 묻기 좋다.

SELECT name, price
FROM products
WHERE price > 800 AND stock >= 10
ORDER BY price ASC;

문제. 콜라(1200,10), 물(800,20), 초콜릿(1500,8)만 있다고 할 때 어떤 상품이 결과에 나오는가?

정답과 풀이 열기

콜라는 두 조건을 모두 만족한다. 물은 가격이 800이라 `> 800`이 거짓이고, 초콜릿은 stock이 8이라 제외된다. 콜라 한 행이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 내 상품 중 2000원 이하 상품 조회문을 작성한다.
  2. 재고가 가장 많은 순으로 모든 상품을 조회한다.
  3. `AND`와 `OR` 결과가 달라지는 조건 한 쌍을 직접 실행한다.
연습 확인 포인트

상품 목록 화면의 필터 기능은 결국 이런 조건 조회와 연결된다.
