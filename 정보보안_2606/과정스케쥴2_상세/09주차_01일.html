콘솔 밖으로: 화면에 상점 제목 띄우기

이제까지 터미널과 결과표에서 보던 데이터를 브라우저 화면으로 보여 준다. 오늘은 React 앱의 시작점만 만진다.

9주차 1일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • React가 화면을 만드는 역할임을 안다.
  • 제공된 프로젝트를 실행한다.
  • JSX로 문장과 가격을 표시한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

React는 값을 화면 모양으로 번역하는 진열 담당

PostgreSQL은 상품을 저장했고 Python은 계산했다. React는 사용자가 클릭하고 보는 화면을 담당한다.

처음부터 설정 명령을 외우지 않는다. 교사가 준비한 실행 가능한 기본 폴더에서 `App.jsx`의 보이는 부분만 직접 바꾼다.

데이터상점명→AppJSX→브라우저제목 표시

손가락 실습: 종이에 DB 창고, API 배송로, React 진열대 그림을 그리고 오늘은 진열대 표지판만 만드는 날임을 표시한다.

직접 따라치기

교사가 제공한 React 기본 프로젝트에서 `src/App.jsx` 내용을 아래처럼 작성한다.

코드를 입력한 뒤 반드시 실행한다

export default function App() {
  const shopName = "우리 반 상점";
  const message = "필요한 학용품을 골라 보세요.";

  return (
    <main>
      <h1>{shopName}</h1>
      <p>{message}</p>
      <p>이번 주 목표: 상품 카드 만들기</p>
    </main>
  );
}

실행 결과 예

브라우저 화면: 우리 반 상점 필요한 학용품을 골라 보세요. 이번 주 목표: 상품 카드 만들기

한 줄씩 읽어 보기

  1. 함수 `App`은 화면 조각을 반환한다.
  2. 중괄호 `{shopName}`은 JavaScript 값을 화면 문장 안에 넣는다.
  3. 저장하면 개발 서버가 변경된 화면을 다시 보여 준다.
입력한 뒤 바꾸어 볼 값

shopName과 message를 자기 상점 문구로 바꾸고 브라우저에 바로 반영되는지 확인한다.

손으로 추적하는 문제

React 자체는 기사 핵심 코드 범위보다 프로젝트 경험용이다. 대신 JavaScript 표현식의 값 흐름을 추적한다.

const price = 1200;
const count = 2;
const text = price * count + "원";
// 화면에는 {text}를 표시한다.

문제. `text`에 들어 있는 문자열은 무엇인가?

정답과 풀이 열기

먼저 숫자 곱셈 2400이 계산되고 문자열 `원`이 붙어 `2400원`이 된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. `today` 변수를 만들어 오늘의 추천 상품 문구에 표시한다.
  2. 숫자 가격 변수를 화면에 `1200원` 형태로 표시한다.
  3. React가 DB 대신 화면을 담당한다는 역할표를 작성한다.
