총액에 따라 할인 적용하기

할인 전 총액이 기준 금액 이상이면 일정 금액을 빼는 규칙을 함수로 만든다.

6주차 3일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 금액 조건을 함수로 분리한다.
  • 할인액과 결제액을 구분한다.
  • 경계 금액을 테스트한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

할인은 장바구니를 다 채운 뒤 찍는 쿠폰 도장

상품마다 마음대로 할인하는 것이 아니라 전체 total이 기준을 넘는지 확인한 뒤 discount를 결정한다.

결제액은 `total - discount`이다. 할인 전/할인액/결제액 세 값을 화면에 따로 보여 주면 계산이 보인다.

총액12000→기준>=10000→할인액1000→결제액11000

손가락 실습: 10000원 경계선을 바닥에 표시하고 총액 카드 9999, 10000, 12000을 어느 쪽에 놓을지 판단한다.

직접 따라치기

함수를 단독 실행하여 경계값의 반환을 먼저 확인한다.

코드를 입력한 뒤 반드시 실행한다

def get_discount(total):
    if total >= 10000:
        return 1000
    return 0

total = 12000
discount = get_discount(total)
payment = total - discount
print("총액:", total)
print("할인:", discount)
print("결제:", payment)

실행 결과 예

총액: 12000 할인: 1000 결제: 11000

한 줄씩 읽어 보기

  1. 함수에 total=12000이 들어가 조건이 참이므로 1000을 반환한다.
  2. 반환된 할인액은 원래 total을 자동으로 바꾸지 않는다.
  3. 결제액을 계산하는 줄에서 실제 차감이 이루어진다.
입력한 뒤 바꾸어 볼 값

total을 9999와 10000으로 바꾸어 할인액이 달라지는 정확한 경계를 확인한다.

손으로 추적하는 문제

경계값에서 `>=`와 `>`의 차이는 시험에도 프로젝트 버그에도 중요하다.

def sale(total):
    if total > 5000:
        return total - 500
    return total
print(sale(5000), sale(5001))

문제. 두 출력값은 무엇인가?

정답과 풀이 열기

5000은 `>` 조건이 거짓이라 5000 그대로이고, 5001은 500을 빼 4501이다. `5000 4501`.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 5000원 이상 500원 할인 규칙 함수를 작성한다.
  2. 할인율 대신 고정 할인액을 쓰는 이유를 초보자 입장에서 적는다.
  3. 어제의 장바구니 total에 함수를 연결한다.
