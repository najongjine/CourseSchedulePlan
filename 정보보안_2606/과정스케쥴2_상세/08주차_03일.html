JOIN으로 흩어진 표 다시 읽기

주문자, 상품명, 수량은 서로 다른 테이블에 있으므로 키를 기준으로 연결해 조회한다.

8주차 3일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • INNER JOIN의 연결 조건을 읽는다.
  • 세 테이블 결과를 표시한다.
  • 잘못된 연결의 문제를 안다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

JOIN은 같은 번호를 가진 카드끼리 옆으로 붙이는 작업

order_items의 order_id와 orders의 order_id가 같을 때 주문자 이름을 옆에 붙인다.

product_id가 같을 때 products의 상품명을 붙이면 사람이 읽을 수 있는 주문 내역이 된다.

orders#1 민지→order_items#1 상품1 x2→products상품1 콜라→결과민지 콜라 2

손가락 실습: 세 종류 카드에서 같은 번호를 실로 이어 하나의 결과 줄을 만든다.

직접 따라치기

JOIN 조건에 사용되는 키를 소리 내 읽은 뒤 SQL을 실행한다.

코드를 입력한 뒤 반드시 실행한다

SELECT o.customer_name, p.name, i.quantity, i.unit_price
FROM orders AS o
JOIN order_items AS i ON o.order_id = i.order_id
JOIN products AS p ON p.product_id = i.product_id
ORDER BY o.order_id, p.name;

실행 결과 예

customer_name | name | quantity | unit_price ---------------+------+----------+----------- 민지 | 물 | 1 | 800 민지 | 콜라 | 2 | 1200

한 줄씩 읽어 보기

  1. `AS o`, `AS i`, `AS p`는 테이블을 짧은 이름으로 부르는 별칭이다.
  2. 첫 JOIN은 주문자와 상세 줄을 주문 번호로 연결한다.
  3. 둘째 JOIN은 상품 번호를 상품명으로 읽을 수 있게 붙인다.
입력한 뒤 바꾸어 볼 값

결과 열에 `i.quantity * i.unit_price AS subtotal`을 추가하고 각 줄 금액을 확인한다.

손으로 추적하는 문제

JOIN 문제는 어떤 열끼리 같아야 올바른 연결인지 선으로 표시하고 푼다.

SELECT p.name, i.quantity
FROM products p
JOIN order_items i ON p.product_id = i.product_id
WHERE i.quantity >= 2;

문제. 이 조회가 보여 주는 것은 무엇인가?

정답과 풀이 열기

주문상세 중 수량이 2개 이상인 줄만 골라, 연결된 상품 이름과 수량을 보여 준다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 세 테이블 연결 결과에 주문 번호도 출력한다.
  2. 주문자 이름이 특정 학생인 행만 조회한다.
  3. JOIN 조건에서 product_id 대신 order_id를 잘못 연결하면 왜 의미가 틀리는지 말한다.
연습 확인 포인트
