INSERT로 상품 행 넣기

테이블에 상품을 한 건씩 등록하고 SELECT로 저장된 결과를 확인한다.

7주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • INSERT 문을 작성한다.
  • 자동 생성된 키를 구분한다.
  • SELECT 결과 표를 읽는다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

INSERT는 빈 표에 상품 카드를 한 줄 끼워 넣는 동작

DB는 프로그램이 끝나도 행을 기억한다. 입력 SQL을 실행한 뒤 조회해야 실제로 어떤 값이 저장됐는지 보인다.

product_id는 DB가 정해 주므로 INSERT 열 목록에서 빼고 name, price, stock만 제공한다.

INSERT콜라/1200/10→DB 번호1→행1 콜라 1200 10→SELECT표시

손가락 실습: 어제 만든 빈 표 그림에 상품 카드 세 장을 한 행씩 추가한다.

직접 따라치기

세 상품을 입력한 뒤 모든 열을 조회한다.

코드를 입력한 뒤 반드시 실행한다

INSERT INTO products (name, price, stock)
VALUES ('콜라', 1200, 10);

INSERT INTO products (name, price, stock)
VALUES ('물', 800, 20);

INSERT INTO products (name, price, stock)
VALUES ('초콜릿', 1500, 8);

SELECT product_id, name, price, stock
FROM products;

실행 결과 예

product_id | name | price | stock ------------+--------+-------+------ 1 | 콜라 | 1200 | 10 2 | 물 | 800 | 20 3 | 초콜릿 | 1500 | 8

한 줄씩 읽어 보기

  1. INSERT 열 순서와 VALUES 값 순서가 맞아야 한다.
  2. ID는 지정하지 않았지만 각 행에 서로 다른 번호가 생성된다.
  3. SELECT는 저장된 행을 읽을 뿐 값을 바꾸지 않는다.
입력한 뒤 바꾸어 볼 값

노트와 펜 상품을 두 건 더 INSERT하고 조회 결과의 행 수를 예상한다.

손으로 추적하는 문제

SQL 결과 문제는 어떤 행이 생성되고 어떤 열이 표시되는지 표로 그린다.

INSERT INTO products (name, price, stock)
VALUES ('펜', 500, 4);

SELECT name, price
FROM products;

문제. 마지막 SELECT 결과에 보이는 열은 무엇이며, 새 행에는 어떤 값이 저장되는가?

정답과 풀이 열기

결과 화면에는 `name`, `price` 두 열만 보인다. 저장된 새 상품 행은 펜, 500, 재고 4와 자동 생성된 product_id를 가진다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 직접 선택한 쇼핑몰 상품을 총 10개 등록한다.
  2. 가격과 재고를 잘못 바꾸어 입력한 행이 없는지 전체 조회한다.
  3. 조회 열 목록을 바꾸어 상품명만 표시한다.
연습 확인 포인트

SQL을 실행할 때마다 INSERT가 다시 실행되면 중복 상품이 생긴다. 실행 구간을 구분하는 습관을 들인다.
