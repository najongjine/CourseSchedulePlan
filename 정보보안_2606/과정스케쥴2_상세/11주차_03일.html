먼저 감각으로 이해하기

주문 요청은 영수증 머리와 상품 줄을 한 봉투에 담은 것

customer_name은 orders 행에 들어갈 정보이고, cart 항목은 order_items 행들이 된다.

장바구니가 비었거나 이름이 비어 있으면 주문을 보내지 않는다.

이름민지→cart상품 두 개→JSON주문 봉투→API저장 요청

손가락 실습: 8주차 영수증 표 그림을 다시 꺼내 화면 JSON에서 어느 값이 어느 표로 갈지 색으로 연결한다.

직접 따라치기

오늘은 요청 객체를 콘솔에 표시하고 검증 규칙을 완성한다.

코드를 입력한 뒤 반드시 실행한다

const [customerName, setCustomerName] = useState("");

function prepareOrder() {
  if (!customerName.trim() || cart.length === 0) {
    alert("이름과 상품을 확인하세요.");
    return;
  }

  const order = {
    customer_name: customerName,
    items: cart.map(item => ({
      product_id: item.product_id,
      quantity: 1,
      unit_price: item.price
    }))
  };
  console.log(order);
}

실행 결과 예

민지, 노트와 펜 담기 후 실행: {customer_name:'민지', items:[{product_id:..., quantity:1, unit_price:...}, ...]}

한 줄씩 읽어 보기

  1. 빈 문자열의 공백을 제거한 뒤 값이 없으면 요청하지 않는다.
  2. cart의 각 상품은 DB order_items에 필요한 값으로 변환된다.
  3. 상품명 대신 product_id를 보내야 DB의 상품 행과 연결할 수 있다.
입력한 뒤 바꾸어 볼 값

상품명이 아닌 product_id가 저장 연결에 필요한 이유를 8주차 외래키 그림으로 설명한다.

손으로 추적하는 문제

프로젝트 데이터 구조는 SQL의 orders/order_items 관계를 이해했을 때 읽을 수 있다.

-- items 한 줄이 아래처럼 저장된다고 가정
INSERT INTO order_items (order_id, product_id, quantity, unit_price)
VALUES (5, 2, 1, 1000);

문제. 이 행은 어떤 내용을 뜻하는가?

정답과 풀이 열기

주문 번호 5에 상품 번호 2를 1개, 당시 단가 1000원으로 담은 상세 한 줄이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 이름 없이 주문 버튼을 눌렀을 때 막히는지 확인한다.
  2. 장바구니 없이 주문할 때 막히는지 확인한다.
  3. 유효한 요청 객체의 items 수가 담은 상품 수와 같은지 확인한다.
