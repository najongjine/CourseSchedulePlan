품절 상태와 버튼 클릭 표시

재고가 없는 상품은 담을 수 없다는 표시를 하고, 버튼 클릭 시 선택 상품을 알려 준다.

9주차 4일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 조건부 표시를 작성한다.
  • 클릭 이벤트 함수를 연결한다.
  • 데이터 상태와 화면 반응을 구분한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

화면은 데이터의 상태를 보여 주는 표정

stock이 0이라는 값이 있으면 화면은 버튼을 막고 품절이라고 알려야 한다.

오늘 클릭은 장바구니 저장 전 단계로, 버튼을 눌렀다는 반응만 확인한다.

stock0→조건stock === 0→버튼disabled→사용자품절 확인

손가락 실습: 상품 카드의 재고 숫자에 따라 초록 담기 카드와 회색 품절 카드를 나누어 놓는다.

직접 따라치기

ProductCard에 클릭 함수와 조건을 추가한다.

코드를 입력한 뒤 반드시 실행한다

function ProductCard({ name, price, stock }) {
  function handleAdd() {
    alert(name + " 상품을 선택했습니다.");
  }

  return (
    <article className="product-card">
      <h2>{name}</h2>
      <p>{price}원</p>
      <p>{stock === 0 ? "품절" : "재고 " + stock + "개"}</p>
      <button onClick={handleAdd} disabled={stock === 0}>
        {stock === 0 ? "담을 수 없음" : "담기"}
      </button>
    </article>
  );
}

실행 결과 예

재고가 있는 카드는 담기 버튼 클릭 시 알림 표시 재고 0 카드는 품절/담을 수 없음 표시와 비활성 버튼

한 줄씩 읽어 보기

  1. 삼항 연산자는 조건이 참일 때 앞 문구, 거짓일 때 뒤 문구를 고른다.
  2. `disabled={stock === 0}`은 품절일 때 버튼을 누를 수 없게 한다.
  3. 클릭 함수는 상품명 props를 읽어 반응한다.
입력한 뒤 바꾸어 볼 값

재고 0인 파일 카드와 재고 1인 파일 카드의 화면 차이를 비교한다.

손으로 추적하는 문제

프로젝트용 JavaScript에서도 조건의 참/거짓을 판별하는 연습은 기사 언어 문제와 공통이다.

const stock = 0;
const label = stock === 0 ? "품절" : "판매중";
const disabled = stock === 0;

문제. `label`과 `disabled`에는 무엇이 들어가는가?

정답과 풀이 열기

stock이 0과 정확히 같으므로 label은 `품절`, disabled는 참(`true`)이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 품절 상품과 판매 상품을 각각 하나씩 화면에 표시한다.
  2. 버튼 문구를 내 문구로 바꿔 본다.
  3. 클릭은 아직 주문 저장이 아니라는 점을 데이터 흐름 그림에 표시한다.
