계산을 함수로 포장하기

주문 금액 계산을 함수로 분리하여 입력과 계산의 역할을 구분한다.

오늘의 목표

  • 함수의 매개변수와 반환값을 구분한다.
  • 소계 계산 함수를 작성한다.
  • 호출된 함수의 값을 추적한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

함수는 값을 받아 결과를 내놓는 계산 창구

함수에 가격과 수량을 건네면 함수 안의 지역 변수로 계산하고 결과를 돌려준다.

프로그램이 길어질수록 계산 규칙을 함수 한곳에 두면 읽고 고치기 쉬워진다.

호출calculate(2500,3)→함수 안2500*3→return7500→total7500

손가락 실습: 가격 카드와 수량 카드를 창구 역할 학생에게 주고 계산 결과 카드만 되돌려 받는다.

직접 따라치기

함수 선언을 `main` 위에 두고, 반환값을 변수에 담아 출력한다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int calculate_price(int price, int quantity) {
    int subtotal = price * quantity;
    return subtotal;
}

int main(void) {
    int prices[3] = {3000, 2500, 2000};
    int menu = 2;
    int quantity = 3;
    int total = calculate_price(prices[menu - 1], quantity);

    printf("주문 금액: %d원\n", total);
    return 0;
}

실행 결과 예

주문 금액: 7500원

한 줄씩 읽어 보기

  1. `prices[menu - 1]`가 먼저 계산되어 2500이 함수로 전달된다.
  2. 함수의 `subtotal`은 함수 안에서만 사용하는 상자다.
  3. `return subtotal`의 7500이 `total` 상자에 저장된다.
입력한 뒤 바꾸어 볼 값

함수 안에서 100원을 할인하도록 `return subtotal - 100;`으로 변경하고 결과를 예측한다.

손으로 추적하는 문제

함수 문제는 전달되는 인자값, 함수 내부 변화, 반환된 값의 저장 위치를 세 단계로 나눠 적는다.

int change(int n) {
    n = n + 5;
    return n * 2;
}
int main(void) {
    int a = 3;
    int b = change(a);
    printf("%d %d", a, b);
}

문제. `a`와 `b`는 각각 무엇이 출력되는가?

정답과 풀이 열기

함수에는 a의 값 3이 복사되어 전달된다. 함수 결과는 16이므로 b=16이고 원래 a는 3 그대로다. 출력은 `3 16`.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. `add_tax(price)`처럼 가격에 10%를 더하는 정수 함수 하나를 만든다.
  2. 가격과 수량을 세 가지로 바꾸어 함수 반환값을 표에 기록한다.
  3. 함수에 전달된 값이 원래 변수 자체인지 복사된 값인지 오늘 예제로 설명한다.
연습 확인 포인트

이번 단계에서는 포인터가 없으므로 함수가 바꾼 매개변수는 원래 `a`를 바꾸지 않는다.
