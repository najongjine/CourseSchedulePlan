먼저 감각으로 이해하기

오늘 화면의 상품은 임시 목록, 다음 주에는 DB 목록

지금은 `products` 배열이 파일 안에 직접 적혀 있다. 화면 구조를 배우기에는 충분하지만 새로고침해도 DB 변경과 연결되지 않는다.

다음 주에는 배열을 PostgreSQL에서 API로 받은 결과로 교체한다. 카드 컴포넌트는 거의 그대로 재사용한다.

샘플 배열오늘→ProductCard재사용→API 응답다음 주→화면같은 카드

손가락 실습: 오늘 배열에 적힌 상품과 7주차 DB 상품 표 사이에 다음 주 화살표를 그린다.

직접 따라치기

상품 데이터와 카드 코드를 한 파일로 합친 필수 버전이다. CSS는 교사 제공 파일을 사용한다.

코드를 입력한 뒤 반드시 실행한다

function ProductCard({ name, price, stock }) {
  return (
    <article className="product-card">
      <h2>{name}</h2>
      <p>{price.toLocaleString()}원</p>
      <p>{stock === 0 ? "품절" : `재고 ${stock}개`}</p>
      <button disabled={stock === 0}>담기</button>
    </article>
  );
}

const products = [
  { id: 1, name: "노트", price: 2000, stock: 5 },
  { id: 2, name: "펜", price: 1000, stock: 8 },
  { id: 3, name: "파일", price: 1500, stock: 0 },
  { id: 4, name: "스티커", price: 500, stock: 12 }
];

export default function App() {
  return (
    <main>
      <h1>우리 반 상점</h1>
      <section className="products">
        {products.map(product => (
          <ProductCard key={product.id} {...product} />
        ))}
      </section>
    </main>
  );
}

실행 결과 예

우리 반 상점 제목 아래 상품 카드 4장 표시 파일 상품만 품절 버튼으로 표시

한 줄씩 읽어 보기

  1. 네 상품 객체가 map을 통과하며 같은 ProductCard 모양 네 장이 된다.
  2. `{...product}`는 객체의 name, price, stock을 props로 전달하는 짧은 표기다.
  3. 품절 조건은 각 카드가 받은 stock 값으로 판단한다.
입력한 뒤 바꾸어 볼 값

펼침 문법이 어렵다면 `name={product.name}` 방식으로 다시 적어 보고 같은 결과인지 확인한다.

손으로 추적하는 문제

React는 기사 직접 범위보다 통합 경험을 위한 도구다. 배열/조건/함수 감각은 계속 활용한다.

const products = [
  { name: "A", stock: 0 },
  { name: "B", stock: 2 }
];
const available = products.filter(p => p.stock > 0);

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 상품 네 장이 표시되는지 확인한다.
  2. 품절 상품 버튼이 눌리지 않는지 확인한다.
  3. 샘플 배열을 API 응답으로 바꿀 위치에 표시를 붙인다.
