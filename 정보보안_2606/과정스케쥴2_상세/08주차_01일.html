주문을 왜 두 표로 나누는가

상품 여러 개가 한 주문에 들어갈 수 있으므로 orders와 order_items 관계를 설계한다.

8주차 1일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결

먼저 감각으로 이해하기

영수증 머리와 상품 줄은 서로 다른 정보

주문자와 주문 시간은 영수증 전체에 한 번만 적지만, 상품과 수량은 여러 줄 들어간다.

그래서 주문 머리 `orders`와 상품 줄 `order_items`를 나누고 같은 order_id로 묶는다.

orders주문 #1 민지→order_id1→order_items콜라2 / 물1→products상품 정보

손가락 실습: 영수증 한 장을 그리고 상단 정보와 구매 품목 줄을 색으로 나누어 어떤 표에 들어갈지 표시한다.

직접 따라치기

7주차 products가 이미 있다고 보고 주문 테이블 두 개를 추가한다.

코드를 입력한 뒤 반드시 실행한다

CREATE TABLE orders (
    order_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_name VARCHAR(50) NOT NULL,
    ordered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE order_items (
    order_item_id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_id INTEGER NOT NULL REFERENCES orders(order_id),
    product_id INTEGER NOT NULL REFERENCES products(product_id),
    quantity INTEGER NOT NULL,
    unit_price INTEGER NOT NULL
);

실행 결과 예

한 줄씩 읽어 보기

  1. orders의 한 행은 주문 전체 한 건이다.
  2. order_items에는 같은 order_id가 여러 번 나타날 수 있다.
  3. `REFERENCES`는 존재하는 주문과 상품 번호에만 연결하라는 규칙이다.
입력한 뒤 바꾸어 볼 값

한 주문에 상품 두 개가 담긴 그림을 그리고 order_items 행이 몇 개인지 표시한다.

손으로 추적하는 문제

기사 대비에서는 기본키가 한 행을 식별하고 외래키가 다른 표의 행을 참조한다는 설명이 필수다.

CREATE TABLE child (
    child_id INTEGER PRIMARY KEY,
    product_id INTEGER REFERENCES products(product_id)
);

문제. `product_id`가 외래키라는 것은 어떤 규칙을 뜻하는가?

정답과 풀이 열기

child의 product_id 값은 products 테이블에 실제로 존재하는 product_id를 가리켜야 한다는 뜻이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 주문 한 건과 상품 세 줄이 연결되는 그림을 그린다.
  2. unit_price를 order_items에 저장하는 이유를 상품 가격이 나중에 바뀌는 상황으로 설명한다.
  3. 테이블 생성 SQL을 직접 입력해 실행한다.
연습 확인 포인트

주문 당시 가격을 남겨야 나중 상품 가격이 바뀌어도 과거 결제 금액을 다시 계산할 수 있다.
