POST와 INSERT로 상품 저장하기

등록 폼의 상품 객체를 API로 보내고, API가 PostgreSQL에 INSERT하도록 연결한다.

10주차 4일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • POST 요청의 목적을 안다.
  • 서버 INSERT SQL을 읽고 작성한다.
  • 저장 후 목록을 다시 조회한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

등록은 화면 카드가 아니라 DB 창고에 새 상자를 넣는 일

화면에서만 상품을 추가하면 새로고침 뒤 사라진다. API가 INSERT를 실행해야 DB에 남는다.

학생은 React의 fetch POST와 서버 함수 안의 표준 INSERT 문장을 핵심으로 입력한다. DB 연결 설정은 제공한다.

폼상품 객체→POST/products→INSERTproducts 행→GET새 목록

손가락 실습: 상품 등록 카드를 API 창구를 통해 DB 표에 넣고, 다시 목록 요청으로 진열대에 올리는 그림을 그린다.

직접 따라치기

서버 함수의 핵심 SQL과 React 제출 코드의 핵심 호출을 함께 본다.

코드를 입력한 뒤 반드시 실행한다

# API 함수 안에서 학생이 확인할 핵심 SQL
sql = '''
    INSERT INTO products (name, price, stock)
    VALUES (%s, %s, %s)
'''
cursor.execute(sql, (name, price, stock))

// React handleSubmit 안의 핵심 요청
await fetch("http://localhost:8000/products", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(newProduct)
});
loadProducts(); // 저장 후 GET으로 목록 다시 읽기

실행 결과 예

등록 폼에서 노트/2000/5 입력 -> DB products 새 행 생성 -> 상품 카드 목록에 노트 표시

한 줄씩 읽어 보기

  1. SQL VALUES 자리는 입력값을 안전하게 전달하는 매개변수 방식으로 채운다.
  2. POST 성공 뒤 목록을 다시 조회해야 DB가 만든 product_id까지 화면에 반영된다.
  3. React 상태만 바꾼 것이 아니라 DB 행이 늘어났는지 SQL SELECT로 확인한다.
입력한 뒤 바꾸어 볼 값

등록 전후 `SELECT COUNT(*) FROM products;` 결과를 기록한다.

손으로 추적하는 문제

입력 문자열을 SQL 문장에 직접 이어 붙이지 않고 매개변수로 전달하는 것은 SQL Injection 예방의 기본이다.

-- 직접 연결하지 않는 안전한 작성 방향
INSERT INTO products (name, price, stock)
VALUES (?, ?, ?);
-- 실제 자리표시는 드라이버 방식에 따라 달라질 수 있다.

문제. 왜 입력한 상품명을 SQL 문자열에 그대로 더해 만들지 않는가?

정답과 풀이 열기

입력 내용이 SQL 명령으로 해석되는 공격과 문법 오류를 막기 위해 값은 매개변수로 전달한다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 상품 한 건을 폼으로 등록하고 DB 조회로 확인한다.
  2. 빈 상품명이나 음수 값은 요청 전 막는 조건을 추가한다.
  3. React -> API -> INSERT -> GET -> React 흐름을 다섯 화살표로 적는다.
