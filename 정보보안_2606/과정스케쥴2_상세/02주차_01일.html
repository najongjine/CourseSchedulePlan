같은 종류의 값을 줄 세우는 배열

분식집 메뉴 세 개의 가격을 별도 변수 세 개가 아니라 배열 한 줄로 저장한다.

2주차 1일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 배열과 인덱스를 그림으로 이해한다.
  • `for`로 배열을 순회한다.
  • 가격 목록을 출력한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

배열은 번호가 붙은 사물함 줄

가격이 세 개일 때 `price1`, `price2`, `price3`로 만들 수도 있지만, 반복해서 다루기 어렵다.

`prices[0]`은 첫 칸이다. C의 배열 번호는 0부터 시작한다는 점이 코드 추적의 핵심이다.

prices[0]3000→prices[1]2500→prices[2]2000

손가락 실습: 의자 세 개에 0,1,2 번호를 붙이고 가격 카드를 올린 후 `prices[1]`을 말하면 해당 카드를 든다.

직접 따라치기

가격 세 개를 배열에 넣고 반복 출력한다. 인덱스와 메뉴 번호가 다를 수 있음을 관찰한다.

코드를 입력한 뒤 반드시 실행한다

#include <stdio.h>

int main(void) {
    int prices[3] = {3000, 2500, 2000};
    int i;

    for (i = 0; i < 3; i++) {
        printf("%d번 칸 가격: %d원\n", i, prices[i]);
    }
    return 0;
}

실행 결과 예

0번 칸 가격: 3000원 1번 칸 가격: 2500원 2번 칸 가격: 2000원

한 줄씩 읽어 보기

  1. 배열에는 세 칸만 있으므로 쓸 수 있는 인덱스는 0, 1, 2이다.
  2. `i++`마다 다음 칸을 읽는다.
  3. `i < 3`이 거짓인 i=3에서는 `prices[3]`을 읽지 않는다.
입력한 뒤 바꾸어 볼 값

가격을 `{3500, 3000, 1500}`으로 바꾸고, `i <= 3`으로 잘못 쓰면 왜 위험한지 설명한다.

손으로 추적하는 문제

배열 문제는 인덱스 시작과 반복 종료 조건을 먼저 표시하면 실수가 줄어든다.

int a[4] = {2, 4, 6, 8};
int sum = 0;
for (int i = 1; i < 4; i++) {
    sum += a[i];
}
printf("%d", sum);

문제. 어떤 칸들이 더해지며 결과는 무엇인가?

정답과 풀이 열기

i는 1,2,3이므로 4 + 6 + 8만 더한다. `a[0]`의 2는 빠지고 결과는 `18`이다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 음료 가격 4개를 배열로 만들고 모두 출력한다.
  2. 가격 배열의 모든 값을 합산하는 반복문을 작성한다.
  3. 인덱스가 0부터 시작하는 이유를 오늘 코드 예로 설명한다.
연습 확인 포인트

총합은 `sum` 상자를 0으로 놓고 각 사물함 값을 하나씩 넣어 커지게 하면 된다.
