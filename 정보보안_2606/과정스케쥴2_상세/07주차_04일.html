UPDATE와 DELETE를 안전하게 사용하기

가격이나 재고를 수정하고 판매하지 않을 상품을 삭제하되, 조건 없는 변경의 위험을 이해한다.

7주차 4일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • UPDATE와 DELETE 문을 작성한다.
  • WHERE 없는 변경을 위험으로 인식한다.
  • 변경 전후를 SELECT로 확인한다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

수정과 삭제는 연필이 아니라 도장: 실행하면 저장값이 바뀐다

SELECT는 보기지만 UPDATE와 DELETE는 데이터 자체를 바꾼다.

따라서 먼저 같은 WHERE로 SELECT하여 대상 행을 확인한 뒤 변경하는 습관을 정한다.

SELECT대상 확인→UPDATEprice 변경→SELECT변경 확인→DELETE행 제거

손가락 실습: 상품 카드에서 바꿀 대상에만 포스트잇을 붙이고, 조건이 없으면 모든 카드가 바뀐다고 보여 준다.

직접 따라치기

상품 번호를 확인한 뒤 한 행의 가격과 재고만 수정한다.

코드를 입력한 뒤 반드시 실행한다

SELECT product_id, name, price, stock
FROM products
WHERE name = '콜라';

UPDATE products
SET price = 1300, stock = 15
WHERE name = '콜라';

SELECT product_id, name, price, stock
FROM products
WHERE name = '콜라';

DELETE FROM products
WHERE name = '판매종료상품';

실행 결과 예

-- 콜라 한 행의 가격/재고만 변경됨 -- 이름이 판매종료상품인 행만 삭제됨

한 줄씩 읽어 보기

  1. 첫 SELECT는 대상이 맞는지 확인하는 안전 절차다.
  2. UPDATE의 SET은 새 값을, WHERE는 어느 행인지를 말한다.
  3. WHERE를 빼면 products의 모든 행 가격과 재고가 같은 값으로 바뀔 수 있다.
입력한 뒤 바꾸어 볼 값

실제 실행 전에 `UPDATE products SET stock = 0;`의 피해를 말로 설명하고 절대 실행하지 않는다.

손으로 추적하는 문제

조건이 붙은 UPDATE/DELETE는 SQL 작성형 문제에서도 빈칸으로 출제하기 좋다.

UPDATE products
SET stock = stock - 1
WHERE product_id = 2;

문제. 이 SQL은 어느 값을 어떻게 바꾸는가?

정답과 풀이 열기

상품 번호가 2인 한 행만 찾아 현재 stock에서 1을 빼 새 stock으로 저장한다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 내가 등록한 상품 하나의 가격을 변경하고 변경 전후를 조회한다.
  2. 재고가 0인 상품을 찾는 SELECT를 먼저 작성한다.
  3. 삭제 SQL 앞에 반드시 확인 SELECT를 두는 이유를 기록한다.
연습 확인 포인트
