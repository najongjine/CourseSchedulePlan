오늘의 목표

  • 지출 가능 조건을 식으로 작성한다.
  • 필드가 아닌 지역 변수로 먼저 계산한다.

2교시예제를 직접 입력하고 실행

4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

조건문은 지갑 문 앞의 확인 직원

지출 금액이 잔액 이하일 때만 돈을 뺀다. 거절된 지출은 잔액을 바꾸면 안 된다.

조건문을 읽을 때는 '어떤 경우에 상자의 값이 바뀌는가'를 표시한다.

잔액5000→지출2000→가능?참→새 잔액3000

손가락 실습: 잔액 카드와 지출 카드 두 장을 비교하여 허락이면 잔액 카드를 새 값으로 교체한다.

직접 따라치기

오늘은 아직 클래스 없이 조건 규칙만 정확히 만든다.

코드를 입력한 뒤 반드시 실행한다

import java.util.Scanner;

public class SpendPractice {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int balance = 5000;

        System.out.print("쓸 금액: ");
        int amount = scanner.nextInt();

        if (amount <= balance) {
            balance = balance - amount;
            System.out.println("지출 완료");
        } else {
            System.out.println("잔액 부족");
        }
        System.out.println("남은 돈: " + balance + "원");
    }
}

실행 결과 예

쓸 금액: 2000 지출 완료 남은 돈: 3000원

한 줄씩 읽어 보기

  1. 2000 <= 5000은 참이므로 뺄셈 문장이 실행된다.
  2. 6000을 입력하면 `else`로 가며 balance는 여전히 5000이다.
  3. 마지막 출력은 어느 길을 지나든 실행된다.
입력한 뒤 바꾸어 볼 값

입력 5000은 허용되는가? `<=`를 `<`로 바꾸면 같은 입력에서 결과가 어떻게 달라지는가?

손으로 추적하는 문제

경계값은 C와 Java 모두 단골 문제다. 정확히 같은 금액을 쓸 수 있는지 조건 기호를 본다.

int balance = 3000;
int amount = 3000;
if (amount < balance) {
    balance -= amount;
} else {
    balance += 100;
}
System.out.print(balance);

문제. 출력은 얼마이며, 지출이 되지 않은 이유는 무엇인가?

정답과 풀이 열기

`amount < balance`는 같은 값일 때 거짓이다. else에서 100을 더하므로 `3100`이 출력된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 입금 기능을 조건문 없이 한 줄로 추가한다.
  2. 음수 지출을 입력하면 돈이 늘어나는 문제를 발견하고 `amount > 0` 조건을 결합한다.
  3. 정상 지출, 잔액과 같은 지출, 잔액 초과 지출을 표로 기록한다.
연습 확인 포인트
