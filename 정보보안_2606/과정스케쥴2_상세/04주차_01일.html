상품 하나를 객체로 표현하기

자판기 상품을 이름, 가격, 재고라는 값 묶음으로 만든다.

4주차 1일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 생성자가 처음 값을 넣는 역할을 이해한다.
  • 상품 객체의 필드를 출력한다.
  • 하나의 현실 대상을 코드 값으로 옮긴다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

상품 객체는 이름표, 가격표, 재고 칸을 가진 작은 상자

음료 하나에는 함께 움직이는 값이 세 개 있다. 클래스는 이 값을 따로 흩어 놓지 않고 한 객체로 묶는다.

생성자는 새 상품을 만드는 순간 이름·가격·재고를 처음 채우는 입고 작업이다.

new Drink콜라,1200,3→name콜라→price1200→stock3

손가락 실습: 실제 음료 그림 아래 이름/가격/재고 포스트잇 세 장을 붙여 한 상품 묶음을 만든다.

직접 따라치기

상품 객체 한 개를 만들고 정보를 출력한다.

코드를 입력한 뒤 반드시 실행한다

class Drink {
    String name;
    int price;
    int stock;

    Drink(String name, int price, int stock) {
        this.name = name;
        this.price = price;
        this.stock = stock;
    }

    void show() {
        System.out.println(name + " " + price + "원 / 재고 " + stock);
    }
}

public class VendingStart {
    public static void main(String[] args) {
        Drink cola = new Drink("콜라", 1200, 3);
        cola.show();
    }
}

실행 결과 예

콜라 1200원 / 재고 3

한 줄씩 읽어 보기

  1. 생성자 매개변수 `name`을 `this.name` 필드에 넣는다.
  2. `cola` 객체 안에는 콜라, 1200, 3이 함께 저장된다.
  3. `show()`는 값을 바꾸지 않고 현재 상태를 출력한다.
입력한 뒤 바꾸어 볼 값

사이다 상품 객체를 하나 더 만들고 가격 1000, 재고 2를 출력한다.

손으로 추적하는 문제

시험에서는 생성자가 필드를 어떻게 초기화하는지, 객체별 값이 무엇인지 물을 수 있다.

class Item {
    int price;
    Item(int price) { this.price = price + 100; }
}
Item a = new Item(500);
Item b = new Item(700);
System.out.print(a.price + b.price);

문제. 출력되는 합계는 얼마인가?

정답과 풀이 열기

a.price는 600, b.price는 800으로 초기화된다. 합계 `1400`이 출력된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 물, 커피 상품 객체를 각각 생성하고 출력한다.
  2. `this.price = price;`를 `price = price;`로 바꿔 결과를 관찰하고 왜 필드가 안 채워지는지 묻는다.
  3. 상품 하나의 필드를 그림으로 그린다.
연습 확인 포인트

`this`는 지금 만들어지는 객체 자신의 칸을 가리킨다.
