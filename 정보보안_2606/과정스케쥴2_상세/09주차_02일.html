ProductCard로 상품 한 장 만들기

상품명, 가격, 재고를 받으면 한 장의 상품 카드가 되는 컴포넌트를 작성한다.

9주차 2일차오늘의 목표4시간 흐름감각으로 이해하기직접 따라치기기사형 코드 추적손으로 푸는 연습금요일 프로젝트 연결
이전다음

오늘의 목표

  • 컴포넌트를 함수로 작성한다.
  • props로 값을 전달한다.
  • 한 상품의 표시 구조를 만든다.

오늘 4시간의 흐름

1교시개념을 그림과 코드로 이해
2교시예제를 직접 입력하고 실행
3교시금요일 작품 기능 한 조각 작성
4교시값을 바꾸고 손 추적 연습

먼저 감각으로 이해하기

컴포넌트는 모양이 같은 상품 카드 틀

콜라 카드와 물 카드는 값은 다르지만 보이는 모양은 같다. 카드 틀 하나를 만들고 서로 다른 값을 건네면 중복을 줄인다.

`props`는 부모 화면이 카드에 건네는 상품 정보 봉투다.

App상품 값 전달→propsname/price/stock→ProductCard카드 틀→화면상품 한 장

손가락 실습: 빈 상품 카드 양식을 한 장 그리고, 이름/가격/재고 포스트잇만 바꿔 여러 상품이 되는 모습을 만든다.

직접 따라치기

같은 `App.jsx`에서 작은 컴포넌트 함수를 위에 작성한다.

코드를 입력한 뒤 반드시 실행한다

function ProductCard({ name, price, stock }) {
  return (
    <article className="product-card">
      <h2>{name}</h2>
      <p>{price}원</p>
      <p>남은 수량: {stock}</p>
      <button>담기</button>
    </article>
  );
}

export default function App() {
  return (
    <main>
      <h1>우리 반 상점</h1>
      <ProductCard name="노트" price={2000} stock={5} />
    </main>
  );
}

실행 결과 예

화면에 노트 / 2000원 / 남은 수량: 5 / 담기 버튼이 카드로 표시됨

한 줄씩 읽어 보기

  1. `ProductCard`는 전달받은 세 값을 자신의 화면 위치에 배치한다.
  2. `price={2000}`은 숫자 값을 전달한다.
  3. 아직 담기 버튼은 보이기만 하고 클릭 기능은 없다.
입력한 뒤 바꾸어 볼 값

같은 컴포넌트를 한 번 더 호출해 펜 상품 카드를 추가한다.

손으로 추적하는 문제

코드 결과를 읽을 때 같은 함수가 서로 다른 인자로 호출된다는 점은 이전 Java 메서드와 닮았다.

function Label({ price }) {
  return <p>{price + 500}원</p>;
}
// <Label price={1000} />

문제. 화면 문구는 무엇인가?

정답과 풀이 열기

price 숫자 1000에 500을 더해 `1500원`이 표시된다.

감각을 굳히는 연습

먼저 종이에 예상 결과를 적고, 그 다음 코드를 바꾸어 실행해서 확인한다. 틀렸다면 틀린 이유가 학습 기록이다.

  1. 상품 카드 두 장을 서로 다른 값으로 표시한다.
  2. stock을 0으로 바꾸고 아직 버튼이 그대로인 것을 관찰한다.
  3. Java Drink 객체의 필드와 ProductCard props의 차이를 적는다.
연습 확인 포인트

props는 표시할 값이며, 다음에는 배열 데이터를 카드 여러 장으로 반복 표시한다.
