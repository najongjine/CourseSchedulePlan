## 1교시: 요구사항은 먼저 그림으로 분해한다

## 1.1 메뉴 프로그램의 요구사항 예

오늘 연습 프로그램의 주제는 금요일과 다른 **온도 기록기**로 한다.

- `1`을 선택하면 오늘의 온도를 한 개 등록한다.
- 입력 가능한 온도는 `-30`부터 `50`까지이다.
- `2`를 선택하면 등록된 온도의 평균, 최고, 최저, 폭염일 수(`30` 이상)를 출력한다.
- `0`을 선택하면 프로그램을 끝낸다.
- 아직 온도가 없는데 `2`를 누르면 안내 문장을 출력한다.

주제는 다르지만 필요한 코드 부품은 점수 통계와 같다.

| 점수 프로젝트 | 온도 연습 프로젝트 | 공통 코드 사고 |
|---|---|---|
| 점수 `0~100` | 온도 `-30~50` | 입력 범위 검사 |
| 합격자 수 | 폭염일 수 | 조건을 만족한 횟수 |
| 최고/최저 점수 | 최고/최저 온도 | 최초 입력 초기화 후 비교 |
| 평균 점수 | 평균 온도 | 합계 / 유효 개수 |

## 1.2 순서도: 메뉴를 먼저 그린다

```text
┌------------------┐
│ sum=0, count=0    │
└---------┬--------┘
          v
┌------------------┐
│ 메뉴 출력/입력    │<-----------------------------┐
└---------┬--------┘                              │
          |                                       │
     ┌----┴----┬----------┐                      │
     v         v          v                       │
 menu==1    menu==2     menu==0                   │
 온도입력    통계출력      종료                    │
 범위검사    count 검사                            │
 누적/비교   평균/최고/최저                        │
     └---------┴----------------------------------┘
```

이 그림을 먼저 그리면, 코드를 쓰면서 "지금 무엇을 해야 하지?" 하고 길을 잃는 일이 줄어든다.

## 1.3 변수를 설계한다

| 변수 | 저장할 값 | 초기값 | 바뀌는 시점 |
|---|---|---:|---|
| `menu` | 사용자가 고른 기능 | 입력받음 | 메뉴를 고를 때마다 |
| `temp` | 방금 입력한 온도 | 입력받음 | 등록 기능에서 |
| `sum` | 유효 온도의 합계 | 0 | 온도 등록 성공 시 |
| `count` | 등록된 온도 수 | 0 | 온도 등록 성공 시 |
| `hot_count` | 30도 이상인 횟수 | 0 | 등록 온도가 30 이상이면 |
| `max_temp` | 최고 온도 | 첫 정상 온도 | 더 높은 온도 등록 시 |
| `min_temp` | 최저 온도 | 첫 정상 온도 | 더 낮은 온도 등록 시 |

코드를 쓰기 전 이 표를 만들 수 있어야 한다. 프로그램 설계는 변수 설계에서 절반이 끝난다.

## 2교시: 메뉴 반복 코드의 뼈대

## 2.1 `while (1)`과 명시적 종료

```c
int menu;

while (1) {
    printf("\n1. 온도 등록  2. 통계 보기  0. 종료\n");
    printf("선택: ");
    scanf("%d", &menu);

    if (menu == 0) {
        printf("프로그램을 종료합니다.\n");
        break;
    } else if (menu == 1) {
        printf("등록 기능\n");
    } else if (menu == 2) {
        printf("통계 기능\n");
    } else {
        printf("없는 메뉴입니다.\n");
    }
}
```

### 흐름 추적: 메뉴 입력 `2, 1, 7, 0`

| 반복 회차 | `menu` | 실행되는 길 | 반복 종료 여부 |
|---:|---:|---|---|
| 1 | 2 | 통계 기능 | 계속 |
| 2 | 1 | 등록 기능 | 계속 |
| 3 | 7 | 없는 메뉴 | 계속 |
| 4 | 0 | 종료 메시지와 `break` | 종료 |

`while (1)`은 조건이 항상 참이라는 뜻이다. 따라서 내부의 `break`가 없다면 끝나지 않는다. 무조건 나쁜 형태는 아니지만, 어디서 종료되는지를 설명할 수 있어야 쓸 수 있다.

## 2.2 온도 등록 기능 결합

```c
} else if (menu == 1) {
    printf("온도: ");
    scanf("%d", &temp);

    if (temp < -30 || temp > 50) {
        printf("입력 범위를 벗어났습니다.\n");
        continue;
    }

    if (count == 0) {
        max_temp = temp;
        min_temp = temp;
    } else {
        if (temp > max_temp) max_temp = temp;
        if (temp < min_temp) min_temp = temp;
    }

    sum += temp;
    count++;

    if (temp >= 30) {
        hot_count++;
    }

    printf("등록되었습니다.\n");
}
```

### 왜 입력 오류 뒤 `continue`인가?

입력 오류가 나면 이 온도는:

- 합계에 더해지면 안 된다.
- 개수가 증가하면 안 된다.
- 최고/최저를 바꾸면 안 된다.
- 폭염일 수에도 들어가면 안 된다.

따라서 오류 메시지를 출력한 뒤 현재 반복의 나머지 처리를 건너뛰고 메뉴로 돌아간다.

## 2.3 통계 보기 기능 결합

```c
} else if (menu == 2) {
    if (count == 0) {
        printf("등록된 온도가 없습니다.\n");
    } else {
        double average = (double)sum / count;
        printf("평균: %.1f\n", average);
        printf("최고: %d, 최저: %d\n", max_temp, min_temp);
        printf("폭염일: %d\n", hot_count);
    }
}
```

`count == 0` 검사는 통계 출력 바로 전에 필요하다. 평균 계산을 먼저 하면 0으로 나누는 잘못된 흐름이 된다.

## 3교시: 복합 코드를 손으로 추적하고 오류를 고친다

## 3.1 상태는 메뉴를 돌아도 남아 있다

온도 입력이 `25`, `32`, `18`이고 그 뒤 통계를 출력한다고 하자.

| 등록 뒤 | `sum` | `count` | `hot_count` | `max_temp` | `min_temp` |
|---|---:|---:|---:|---:|---:|
| 시작 | 0 | 0 | 0 | 미정 | 미정 |
| 25 등록 | 25 | 1 | 0 | 25 | 25 |
| 32 등록 | 57 | 2 | 1 | 32 | 25 |
| 18 등록 | 75 | 3 | 1 | 32 | 18 |

```text
평균: 25.0
최고: 32, 최저: 18
폭염일: 1
```

반복문이 다시 시작된다고 해서 `sum`이 0으로 돌아가면 안 된다. 통계 변수들은 메뉴 반복문 **밖에서 한 번만 초기화**한다.

## 기사형 복합 추적 문제 5개

### 문제 1

```c
int sum = 0, count = 0;
for (int i = 1; i <= 4; i++) {
    int score = i * 25;
    if (score > 100) continue;
    sum += score;
    count++;
}
printf("%d %d %.1f", sum, count, (double)sum / count);
```

### 문제 2

```c
int count = 0;
int menu = 1;
while (menu != 0) {
    count++;
    menu--;
}
printf("%d %d", count, menu);
```

### 문제 3

```c
int max = 0;
int count = 0;
for (int score = -1; score <= 1; score++) {
    if (count == 0 || score > max) max = score;
    count++;
}
printf("%d", max);
```

### 문제 4

```c
int sum = 0;
for (int i = 1; i <= 5; i++) {
    if (i % 2 == 0) continue;
    if (i == 5) break;
    sum += i;
}
printf("%d", sum);
```

### 문제 5

```c
int sum = 0, valid = 0;
int input[4] = {80, -1, 120, 60};
for (int i = 0; i < 4; i++) {
    int score = input[i];
    if (score == -1) break;
    if (score < 0 || score > 100) continue;
    sum += score;
    valid++;
}
printf("%d %d", sum, valid);
```

<details>
<summary>정답과 해설</summary>

| 문제 | 출력 | 이유 |
|---|---|---|
| 1 | `250 4 62.5` | 점수는 25, 50, 75, 100 모두 포함된다. |
| 2 | `1 0` | 첫 반복에서 count 증가, menu가 0이 되어 종료된다. |
| 3 | `1` | 첫 값 -1로 초기화 후 0, 1에서 갱신된다. |
| 4 | `4` | 1과 3만 누적, 5에서는 더하기 전 `break`한다. |
| 5 | `80 1` | `-1`이 종료값이어서 뒤 입력은 보지 않는다. |

</details>

## 3.2 오류 수정 실습

아래 코드는 메뉴 프로그램 일부이다. 점수를 입력한 뒤 통계가 이상해질 수 있는 문제를 찾는다.

```c
int sum = 0;
int count = 0;
int max_score = 0;
int menu;

while (1) {
    scanf("%d", &menu);

    if (menu == 0) {
        continue;
    } else if (menu == 1) {
        int score;
        scanf("%d", &score);

        if (score < 0 && score > 100) {
            printf("오류\n");
        }

        count++;
        sum += score;
        if (score > max_score) max_score = score;
    } else if (menu == 2) {
        printf("%.1f\n", (double)sum / count);
    }
}
```

### 찾아야 할 오류 최소 4개

| 문제 위치 | 잘못된 행동 | 수정 아이디어 |
|---|---|---|
| 종료 메뉴 |  |  |
| 범위 논리식 |  |  |
| 오류 입력 뒤 처리 |  |  |
| 통계 보기 전 검사 |  |  |
| 최고점 초기화 |  |  |

<details>
<summary>오류 수정 포인트</summary>

- `menu == 0`이면 `continue`가 아니라 `break`로 종료해야 한다.
- 범위 밖은 `score < 0 || score > 100`이다. 동시에 작고 클 수는 없다.
- 잘못된 입력이면 누적하지 않도록 메시지 뒤 `continue`하거나 `else` 안에서 누적한다.
- `count == 0`일 때 평균을 계산하지 않아야 한다.
- 점수가 음수도 가능한 다른 프로그램이라면 `max_score = 0`은 안전하지 않다. 첫 정상 입력으로 초기화하는 습관이 좋다.

</details>

## 4교시: 제한 시간 구현 연습

## 미니 과제: 온도 기록기

금요일의 점수 프로그램 답안을 미리 만들지 않기 위해, 오늘은 같은 구조의 다른 주제를 구현한다. `day4_menu.c`를 빈 파일에서 작성한다.

### 요구사항

1. 메뉴는 `1. 온도 등록`, `2. 통계 출력`, `0. 종료`이다.
2. 온도 범위는 `-30~50`이고 범위 밖 입력은 제외한다.
3. 통계는 등록 수, 평균, 최고, 최저, 30도 이상 횟수를 출력한다.
4. 입력이 없을 때 통계를 요청하면 안내 메시지만 출력한다.
5. 잘못된 메뉴 번호에도 안내하고 다시 메뉴를 보여 준다.

### 제한 시간 운영

| 구간 | 시간 | 해야 할 것 |
|---|---:|---|
| 설계 | 10분 | 변수표와 순서도 작성 |
| 구현 1 | 15분 | 메뉴 반복과 종료만 동작 |
| 구현 2 | 15분 | 등록과 범위 검증 |
| 구현 3 | 15분 | 통계 출력 |
| 테스트/수정 | 20분 | 아래 테스트 실행 및 기록 |

### 테스트 시나리오

| 순서 입력 | 목적 | 기대되는 핵심 결과 |
|---|---|---|
| `2 -> 0` | 등록 전 통계 | 등록 없음 안내 후 종료 |
| `1 -> 25 -> 1 -> 32 -> 2 -> 0` | 정상 누적 | 평균 28.5, 최고 32, 폭염일 1 |
| `1 -> 70 -> 2 -> 0` | 오류 값 제외 | 통계 없음 |
| `9 -> 0` | 없는 메뉴 | 안내 후 정상 종료 |
