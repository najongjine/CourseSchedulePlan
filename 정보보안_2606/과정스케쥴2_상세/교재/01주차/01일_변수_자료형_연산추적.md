## 0. 시작 전: 프로그램은 마술 상자가 아니다

처음 코드를 보면 기호가 많아 보인다. 그러나 오늘 볼 프로그램의 일은 세 가지뿐이다.

```text
사용자가 점수를 입력한다     컴퓨터가 값으로 계산한다       결과를 화면에 보여 준다
        입력          ->              처리            ->             출력
     80, 95                       합계 175                     평균 87.5
```

컴퓨터는 "국어 점수"라는 뜻을 스스로 이해하지 않는다. 우리가 `korean`이라는 이름의 공간을 만들고 거기에 `80`을 저장했기 때문에, 뒤에서 그 숫자를 다시 꺼낼 수 있다.

### 첫 번째 시각화: 변수는 이름표가 붙은 상자

```text
int korean = 80;       int english = 95;        int total = korean + english;

┌────────────┐         ┌────────────┐          ┌────────────┐
│ korean     │         │ english    │          │ total      │
│ int : 80   │         │ int : 95   │          │ int : 175  │
└────────────┘         └────────────┘          └────────────┘
```

- `int`는 정수를 저장하는 상자 종류이다.
- `=`는 수학의 "같다"보다 **오른쪽 결과를 왼쪽 상자에 저장하라**는 명령에 가깝다.
- `total = korean + english;`가 실행되어도 `korean`과 `english`의 값은 없어지지 않는다.

### 손으로 먼저 해 보기

종이에 `a`, `b`, `c` 상자를 그린 뒤 아래 코드를 실행 순서대로 채운다.

```c
int a = 10;
int b = a + 3;
a = b * 2;
int c = a - b;
printf("%d %d %d", a, b, c);
```

| 실행 직후 | `a` | `b` | `c` |
|---|---:|---:|---:|
| `int a = 10;` | 10 | - | - |
| `int b = a + 3;` | 10 | 13 | - |
| `a = b * 2;` | 26 | 13 | - |
| `int c = a - b;` | 26 | 13 | 13 |

따라서 출력은 `26 13 13`이다. 이 표를 쓰는 습관이 정보처리기사 코드 추적의 시작이다.

## 1.2 `int`와 `double`: 상자 종류가 결과를 바꾼다

점수는 보통 정수로 입력하지만 평균은 소수일 수 있다.

```text
국어 80 + 영어 95 = 총점 175
총점 175 / 과목 수 2 = 평균 87.5
```

그런데 C에서 정수끼리 나누면 소수 부분을 보관하지 않는다.

```c
int total = 175;
int subject_count = 2;
double wrong_average = total / subject_count;
double correct_average = (double)total / subject_count;
```

```text
total / subject_count          (double)total / subject_count
     175 / 2                         175.0 / 2
        │                               │
   정수끼리 계산                    실수 계산
        │                               │
       87                              87.5
```

`wrong_average`의 상자는 `double`이어도, 이미 오른쪽에서 `87`로 계산된 뒤 저장되므로 `87.0`이 된다.

### 실습 코드: 잘못된 평균과 올바른 평균 비교

```c
#include <stdio.h>

int main(void) {
    int korean = 80;
    int english = 95;
    int total = korean + english;
    double wrong_average = total / 2;
    double correct_average = (double)total / 2;

    printf("total: %d\n", total);
    printf("wrong average: %.1f\n", wrong_average);
    printf("correct average: %.1f\n", correct_average);
    return 0;
}
```

예상 출력:

```text
total: 175
wrong average: 87.0
correct average: 87.5
```

### 교실 질문

- `double wrong_average`인데 왜 `87.5`가 되지 않았는가?
- `(double)`은 상자를 바꾸는가, 계산할 순간의 값을 바꾸는가?
- 점수가 `90`, `90`이면 이 오류를 발견하기 어려운 이유는 무엇인가?

마지막 질문이 중요하다. 오류는 우연히 결과가 같으면 숨어 버린다. 그래서 테스트 데이터는 `80`, `95`처럼 소수가 생기는 값도 포함해야 한다.

## 2교시: 입력, 산술 연산, 나머지

## 2.1 `scanf`: 키보드 값을 변수 상자에 넣기

고정된 `80`, `95` 대신 학생이 입력하도록 바꾼다.

```text
키보드 입력             scanf                 변수 상자
  80  Enter     ->    "%d", &korean    ->     korean: 80
```

`&korean`은 오늘은 **korean 상자의 위치표**라고 이해한다. `scanf`는 값을 저장하려면 상자가 어디 있는지 알아야 한다.

```c
#include <stdio.h>

int main(void) {
    int korean;
    int english;
    int total;
    double average;

    printf("국어 점수: ");
    scanf("%d", &korean);

    printf("영어 점수: ");
    scanf("%d", &english);

    total = korean + english;
    average = (double)total / 2;

    printf("총점: %d\n", total);
    printf("평균: %.1f\n", average);
    return 0;
}
```

### 코드 입력 전 예측표

입력을 국어 `76`, 영어 `89`로 정하고 실행 전 먼저 채운다.

| 단계 | `korean` | `english` | `total` | `average` | 출력 |
|---|---:|---:|---:|---:|---|
| 선언 직후 | ? | ? | ? | ? | - |
| 두 입력 후 | 76 | 89 | ? | ? | 안내문 2개 |
| 총점 계산 후 | 76 | 89 | 165 | ? | - |
| 평균 계산 후 | 76 | 89 | 165 | 82.5 | - |
| 출력 후 | 76 | 89 | 165 | 82.5 | 총점/평균 |

선언만 된 지역 변수는 믿을 수 있는 값이 아직 없으므로 `?`라고 적는다. 입력이나 대입 전에 출력하려 해서는 안 된다.

## 2.2 `%`: 나머지로 규칙 찾기

나머지 연산자 `%`는 시험 코드에서 자주 쓰인다.

| 식 | 결과 | 의미 |
|---|---:|---|
| `10 % 3` | 1 | 10을 3개씩 묶고 1 남음 |
| `8 % 2` | 0 | 짝수 |
| `9 % 2` | 1 | 홀수 |
| `score % 10` | 일의 자리 | 점수 마지막 숫자 |

```c
int score = 87;
printf("%d\n", score / 10);  // 8: 십의 자리 몫
printf("%d\n", score % 10);  // 7: 나머지
```

## 3교시: 연산 순서와 증감 연산 추적

## 3.1 연산 순서

기본적으로 곱셈, 나눗셈, 나머지가 덧셈과 뺄셈보다 먼저 계산된다. 확실히 하고 싶으면 괄호를 쓴다.

```c
int a = 10 + 2 * 3;       // 16
int b = (10 + 2) * 3;     // 36
int c = 20 / 3 + 1;       // 7
double d = (double)20 / 3 + 1; // 7.666...
```

### 시각화: 계산 나무

```text
10 + 2 * 3                    (10 + 2) * 3
     └─ 6                         └─ 12
 └──── 16                             └─ 36
```

## 3.2 증감 연산: `++`가 앞인지 뒤인지

`count++`는 `count = count + 1`처럼 값을 하나 늘린다. 단독 문장일 때는 `++count`와 같은 결과이다.

```c
int count = 0;
count++;
++count;
printf("%d", count);  // 2
```

하지만 다른 계산 안에 함께 들어가면 현재 줄에서 사용할 값이 달라진다.

```c
int a = 3;
int b = a++;  // b에 먼저 3 저장, 그 뒤 a가 4
int c = ++a;  // a를 먼저 5로 증가, c에 5 저장
printf("%d %d %d", a, b, c);
```

### 추적 그림

```text
시작          b = a++                 c = ++a
a: 3     ->   b: 3, a: 4       ->     a: 5, c: 5

최종 출력: a=5, b=3, c=5
```

초보 단계에서 실무 코드에는 증감 연산을 복잡한 식에 섞지 않는 편이 읽기 쉽다. 다만 기사형 코드 읽기에서는 반드시 추적할 수 있어야 한다.

## 4교시: 고쳐 쓰기와 당일 미니 과제

## 오류 수정 실습

아래 프로그램은 두 점수의 평균을 소수 첫째 자리까지 보여 주려 했으나 잘못되어 있다. 오류를 찾아 이유를 적고 고친다.

```c
#include <stdio.h>

int main(void) {
    int first;
    int second;
    int total;
    double average;

    printf("첫 점수: ");
    scanf("%d", first);
    printf("둘째 점수: ");
    scanf("%d", &second);

    total = first + second;
    average = total / 2;

    printf("총점: %d\n", total);
    printf("평균: %d\n", average);
    return 0;
}
```

### 찾아야 할 문제

| 위치 | 무엇이 잘못됐는가 | 고치는 방향 |
|---|---|---|
| 첫 입력 |  |  |
| 평균 계산 |  |  |
| 평균 출력 |  |  |

<details>
<summary>오류 수정 포인트</summary>

- `scanf("%d", first);`는 저장할 위치가 아니므로 `&first`여야 한다.
- `average = total / 2;`는 정수 나눗셈이므로 `average = (double)total / 2;`로 고친다.
- `double` 출력에는 `%d`가 아니라 `%.1f`를 사용한다.

</details>
