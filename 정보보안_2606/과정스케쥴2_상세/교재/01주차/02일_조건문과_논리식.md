## 0. 어제 코드가 오늘의 출발점이다

어제 만든 프로그램은 다음 일을 했다.

```text
세 점수 입력 -> 총점 계산 -> 평균 계산 -> 출력
```

오늘은 흐름을 버리지 않고 중간에 판단을 추가한다.

```text
세 점수 입력 -> 범위 검사 -> 총점/평균 계산 -> 등급 판정 -> 합격 판정 -> 출력
                    ↑                                   ↑
                 조건문                              논리식
```

조건문을 배우는 날이라고 해서 계산을 지우고 `if`만 연습하지 않는다. 시험 코드는 대부분 여러 문법이 이미 섞여 있다.

## 전날 워밍업 손 추적

```c
int korean = 80;
int english = 95;
int math = 78;
int total = korean + english + math;
double average = (double)total / 3;
printf("%d %.1f", total, average);
```

1. `total`은 얼마인가?
2. `average`는 소수 첫째 자리 출력 시 얼마인가?
3. `(double)`을 제거하면 무엇이 달라지는가?

<details>
<summary>워밍업 정답</summary>

`total`은 253이고, 정확한 평균은 `84.3`으로 출력된다. `(double)`을 제거하면 정수 계산 `253 / 3`의 결과 84가 먼저 만들어져 `84.0`이 출력된다.

</details>

## 1교시: 조건은 갈림길이다

## 1.1 참과 거짓

C에서 조건은 질문이다. 예를 들어 `score >= 60`은 "점수가 60 이상인가?"라는 질문이다.

```text
score = 75

score >= 60 ?  ---- 참 ----> "합격" 출력
       |
       └------ 거짓 ------> 아무 일도 하지 않음
```

```c
int score = 75;

if (score >= 60) {
    printf("합격\n");
}
```

### 비교 연산자 표

| 코드 | 읽는 방법 | `score = 60`일 때 결과 |
|---|---|---|
| `score > 60` | 60보다 크다 | 거짓 |
| `score >= 60` | 60 이상이다 | 참 |
| `score < 60` | 60보다 작다 | 거짓 |
| `score <= 60` | 60 이하이다 | 참 |
| `score == 60` | 60과 같다 | 참 |
| `score != 60` | 60과 다르다 | 거짓 |

`=`는 저장이고 `==`는 비교이다. 이 차이는 시험뿐 아니라 실제 오류의 단골 원인이다.

## 1.2 경계값: 딱 걸치는 숫자가 가장 중요하다

"60점 이상 합격"이라는 요구사항은 `59`, `60`, `61`을 시험해야 정확하다.

```text
59점              60점              61점
불합격       |     합격       |      합격
             경계가 갈리는 지점
```

### 실습 1: 어제 입력 코드에 합격 출력 추가

```c
#include <stdio.h>

int main(void) {
    int score;

    printf("점수: ");
    scanf("%d", &score);

    if (score >= 60) {
        printf("합격\n");
    } else {
        printf("불합격\n");
    }

    return 0;
}
```

### 실습 규칙

실행하기 전에 다음 표의 예상 출력부터 채운다.

| 입력 | `score >= 60`의 값 | 예상 출력 | 실제 출력 |
|---:|---|---|---|
| 59 |  |  |  |
| 60 |  |  |  |
| 61 |  |  |  |

## 2교시: 여러 길 중 하나를 선택하는 등급 판정

## 2.1 `else if`는 위에서부터 통과하는 첫 길 하나

등급 요구사항을 읽는다.

| 평균 범위 | 등급 |
|---:|---|
| 90 이상 | A |
| 80 이상 90 미만 | B |
| 70 이상 80 미만 | C |
| 60 이상 70 미만 | D |
| 60 미만 | F |

이를 코드로 옮기면 높은 기준부터 검사하는 것이 자연스럽다.

```c
if (average >= 90) {
    printf("A\n");
} else if (average >= 80) {
    printf("B\n");
} else if (average >= 70) {
    printf("C\n");
} else if (average >= 60) {
    printf("D\n");
} else {
    printf("F\n");
}
```

### 시각화: `average = 83.5`가 지나가는 길

```text
average >= 90 ? 거짓
       |
average >= 80 ? 참  ----------> B 출력, 나머지 조건은 보지 않음
       |
average >= 70 ? (실행되지 않음)
```

`83.5`는 사실 `>= 70`도 참이지만, 이미 `>= 80` 길로 들어갔기 때문에 B만 출력한다.

## 2.2 잘못된 순서가 만드는 버그

```c
if (average >= 60) {
    printf("D\n");
} else if (average >= 70) {
    printf("C\n");
} else if (average >= 80) {
    printf("B\n");
} else if (average >= 90) {
    printf("A\n");
}
```

평균이 `95`이어도 첫 조건 `average >= 60`이 참이라서 `D`가 출력된다. 코드가 컴파일되는 것과 요구사항대로 동작하는 것은 다른 문제다.

### 직접 작성 실습 2: 세 과목 평균 등급

다음 요구사항을 구현한다.

1. 국어, 영어, 수학 점수를 입력받는다.
2. 총점과 정확한 실수 평균을 계산한다.
3. 평균 기준 A~F 등급을 출력한다.
4. 평균을 소수 첫째 자리까지 출력한다.

테스트는 반드시 `90.0`, `89.7`, `80.0`, `59.7` 근처가 되도록 점수를 고른다.

## 3교시: 복합 조건은 요구사항의 문장을 코드로 번역하는 일

## 3.1 `&&`: 둘 다 참이어야 한다

"평균이 60점 이상이고, 모든 과목이 40점 이상이면 합격"이라는 조건을 생각한다.

```text
평균 >= 60        모든 과목 >= 40           최종
   참       AND        참             ->    합격
   참       AND        거짓           ->    불합격(과락)
   거짓     AND        참             ->    불합격
```

```c
if (average >= 60 && korean >= 40 && english >= 40 && math >= 40) {
    printf("합격\n");
} else {
    printf("불합격\n");
}
```

## 3.2 `||`: 하나라도 참이면 문제이다

같은 요구사항을 "과락 여부" 관점에서 쓰면 다음과 같다.

```c
if (korean < 40 || english < 40 || math < 40) {
    printf("과락\n");
}
```

```text
korean < 40     english < 40     math < 40
    거짓     OR      참       OR    거짓      -> 과락 있음
```

## 3.3 `!`: 질문의 답을 뒤집는다

```c
int valid = score >= 0 && score <= 100;

if (!valid) {
    printf("올바르지 않은 점수\n");
}
```

`valid`가 참이면 점수가 올바르다. `!valid`는 "올바르지 않다"이므로 입력 오류 메시지를 출력할 조건이 된다.

## 3.4 입력 범위 검사

어제 프로그램은 `130`을 입력해도 평균을 계산했다. 오늘은 계산 전에 막는다.

```c
if (korean < 0 || korean > 100 ||
    english < 0 || english > 100 ||
    math < 0 || math > 100) {
    printf("점수는 0부터 100까지 입력해야 합니다.\n");
    return 0;
}
```

### 흐름 그림

```text
점수 입력
   |
범위 밖 점수가 하나라도 있는가? -- 예 --> 오류 출력 후 종료
   |
  아니오
   v
총점/평균 계산 -> 등급 판정 -> 합격/과락 판정
```

### 직접 작성 실습 3: 합격과 과락을 동시에 출력

실습 2 코드에 다음을 추가한다.

- 평균이 60 이상이고 각 과목이 40 이상이면 `최종 합격`
- 평균은 60 이상이지만 어느 한 과목이 40 미만이면 `평균 통과, 과락 불합격`
- 평균이 60 미만이면 `평균 미달 불합격`
- 입력 범위를 먼저 검사한다.

### 테스트 데이터 설계

| 사례 | 국어 | 영어 | 수학 | 평균 예상 | 판정 예상 | 왜 필요한가 |
|---|---:|---:|---:|---:|---|---|
| 정상 합격 | 70 | 80 | 90 |  |  | 기본 동작 |
| 평균 통과 과락 | 100 | 100 | 39 |  |  | `&&` 확인 |
| 평균 미달 | 40 | 50 | 59 |  |  | 평균 조건 확인 |
| 범위 오류 | 80 | 101 | 70 | 계산 안 함 |  | 검증 순서 확인 |

## 기사형 손 추적 훈련 8문제

### 문제 1

```c
int score = 60;
if (score > 60) printf("A");
else printf("B");
```

### 문제 2

```c
int score = 90;
if (score >= 60) printf("P");
else if (score >= 90) printf("A");
```

### 문제 3

```c
int a = 70, b = 35;
if (a >= 60 && b >= 40) printf("PASS");
else printf("FAIL");
```

### 문제 4

```c
int a = 70, b = 35;
if (a < 40 || b < 40) printf("X");
else printf("O");
```

### 문제 5

```c
int score = 100;
int valid = score >= 0 && score <= 100;
if (!valid) printf("E");
else printf("V");
```

### 문제 6

```c
int total = 269;
double avg = (double)total / 3;
if (avg >= 90) printf("A");
else if (avg >= 80) printf("B");
else printf("C");
```

### 문제 7

```c
int korean = 39, english = 100, math = 100;
double avg = (double)(korean + english + math) / 3;
if (avg >= 60 && korean >= 40 && english >= 40 && math >= 40)
    printf("P");
else
    printf("F");
```

### 문제 8

```c
int x = 5;
if (x++ > 5) printf("%d", x);
else printf("%d", ++x);
```

<details>
<summary>정답과 핵심 해설</summary>

| 문제 | 출력 | 이유 |
|---|---|---|
| 1 | `B` | 60은 `> 60`이 아니다. |
| 2 | `P` | 첫 조건이 이미 참이어서 두 번째 조건은 실행되지 않는다. |
| 3 | `FAIL` | `b >= 40`이 거짓이며 `&&`는 모두 참이어야 한다. |
| 4 | `X` | `b < 40` 하나가 참이므로 `||` 결과는 참이다. |
| 5 | `V` | 100은 범위 안이며 `!valid`는 거짓이다. |
| 6 | `B` | 평균은 약 89.67로 90 미만, 80 이상이다. |
| 7 | `F` | 평균은 높지만 국어 과락이다. |
| 8 | `7` | 비교에는 증가 전 5를 사용해 거짓, 그 뒤 `x=6`; else에서 먼저 증가하여 7 출력. |

</details>

## 4교시: 오류 수정과 누적 미니 과제

## 오류 수정 문제: 왜 모두 장학생이 되는가

다음 코드는 평균 90 이상이고 과락이 없는 학생만 장학생으로 표시하려는 코드이다.

```c
int korean = 95;
int english = 35;
int math = 95;
double average = (double)(korean + english + math) / 3;

if (average >= 90 || korean >= 40 || english >= 40 || math >= 40) {
    printf("장학생\n");
} else {
    printf("일반\n");
}
```

### 작업

1. 현재 출력이 무엇인지 손으로 예측한다.
2. 왜 요구사항과 다른지 논리식의 말로 설명한다.
3. 조건식을 고쳐 네 가지 테스트 데이터를 실행한다.

<details>
<summary>수정 방향</summary>

장학생은 조건을 모두 만족해야 하므로 `average >= 90 && korean >= 40 && english >= 40 && math >= 40`이어야 한다. 주어진 점수의 평균은 75이므로 수정 후 출력은 `일반`이다.

</details>

## 당일 미니 과제: 개인 성적 판정표

`day2_judge.c`를 빈 파일에서 작성한다. 어제 코드를 복사해 시작하지 않고, 필요한 흐름을 먼저 종이에 쓴 다음 작성한다.

### 요구사항

1. 이름 대신 학생 번호 정수 한 개를 입력받는다.
2. 국어, 영어, 수학 점수를 입력받는다.
3. 점수가 하나라도 `0~100` 범위를 벗어나면 오류 메시지를 출력하고 종료한다.
4. 총점, 소수 첫째 자리 평균, A~F 등급을 출력한다.
5. 평균 60 이상이면서 전 과목 40 이상이면 합격이다.
6. 평균은 통과했지만 과락이면 `과락 불합격`을 별도로 출력한다.
7. 평균 90 이상이며 과락이 없으면 `우수 학습자` 메시지도 출력한다.

### 출력 형식 예

```text

국어 영어 수학: 95 88 92
--- 결과표 ---
번호: 12
총점: 275
평균: 91.7
등급: A
판정: 합격
추가: 우수 학습자
```

### 테스트표

| 번호 | 입력 점수 | 확인하려는 조건 | 예상 출력 핵심 | 실제 |
|---:|---|---|---|---|
| 1 | `90 90 90` | A/우수 경계 |  |  |
| 2 | `100 100 39` | 평균 통과 + 과락 |  |  |
| 3 | `60 60 59` | 평균 60 경계 인근 |  |  |
| 4 | `-1 80 80` | 입력 범위 오류 |  |  |
| 5 | `0 0 0` | 범위는 정상, 불합격 |  |  |
