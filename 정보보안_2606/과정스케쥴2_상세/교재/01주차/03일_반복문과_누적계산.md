## 0. 오늘의 큰 그림: 복사는 반복문이 아니다

어제 한 명의 평균을 계산하는 코드를 만들었다. 다섯 명을 처리한다고 코드를 다섯 번 복사하면 길고, 인원 수가 바뀌면 다시 고쳐야 한다.

```text
한 명 처리 로직
점수 입력 -> 범위 확인 -> 평균 계산 -> 합격 판정

여러 명 처리 로직
┌----------------------------------------------------┐
│ 점수 입력 -> 범위 확인 -> 평균 계산 -> 합격 판정   │  x 학생 수
└----------------------------------------------------┘
                     |
          총점/합격자수/최고점에 누적
```

반복문은 "같은 모양의 처리"를 필요한 횟수만큼 다시 실행하는 장치이다. 중요한 점은 반복마다 새 입력을 받고, 누적 변수에는 이전 반복의 결과가 남아 있다는 것이다.

## 전날 복습 문제

다음 한 학생의 판정 결과를 실행 없이 적는다.

```c
int korean = 100, english = 100, math = 39;
int total = korean + english + math;
double average = (double)total / 3;

if (average >= 60 && korean >= 40 && english >= 40 && math >= 40)
    printf("PASS");
else if (average >= 60)
    printf("FAIL-CUT");
else
    printf("FAIL-AVG");
```

</details>

## 1교시: `for` 문은 횟수가 보이는 반복

## 1.1 반복문의 세 칸

```c
for (int i = 1; i <= 5; i++) {
    printf("%d번째 학생\n", i);
}
```

```text
초기화             조건 확인                몸통 실행              변화
i = 1       ->     i <= 5 ? 참      ->      1번째 출력      ->     i = 2
                    i <= 5 ? 참      ->      2번째 출력      ->     i = 3
                         ... 
                    i <= 5 ? 참      ->      5번째 출력      ->     i = 6
                    i <= 5 ? 거짓    ->      종료
```

### 반복 추적표

| 반복 차례 | 몸통 시작 시 `i` | `i <= 5` | 출력 | 증가 후 `i` |
|---:|---:|---|---|---:|
| 1 | 1 | 참 | `1번째 학생` | 2 |
| 2 | 2 | 참 | `2번째 학생` | 3 |
| 3 | 3 | 참 | `3번째 학생` | 4 |
| 4 | 4 | 참 | `4번째 학생` | 5 |
| 5 | 5 | 참 | `5번째 학생` | 6 |
| 종료 확인 | 6 | 거짓 | 없음 | - |

정보처리기사 문제에서 반복문을 보면 먼저 **몸통이 몇 번 실행되는지**를 확정한다. 그 뒤에 몸통 안 계산을 따라간다.

## 1.2 누적 변수는 반복 밖에서 시작한다

```c
int sum = 0;

for (int i = 1; i <= 3; i++) {
    int score;
    scanf("%d", &score);
    sum = sum + score;
}

printf("합계: %d\n", sum);
```

입력이 `70 80 90`일 때:

```text
처음 sum: 0
1회 입력 70 -> sum:  0 + 70 = 70
2회 입력 80 -> sum: 70 + 80 = 150
3회 입력 90 -> sum:150 + 90 = 240
```

### 반드시 구분할 코드

잘못된 코드:

```c
for (int i = 1; i <= 3; i++) {
    int sum = 0;
    int score;
    scanf("%d", &score);
    sum = sum + score;
}
```

`sum`을 반복 안에서 매번 새로 0으로 만들면 과거 입력이 사라진다. 누적할 상자는 반복 바깥에 있어야 한다.

## 2교시: `while`, `do while`, `break`, `continue`

## 2.1 `while`: 조건을 먼저 확인한다

반복 횟수보다 "종료 명령이 올 때까지"가 중요하면 `while`이 자연스럽다.

```c
int score;

printf("점수 입력, 종료는 -1: ");
scanf("%d", &score);

while (score != -1) {
    printf("입력 점수: %d\n", score);
    printf("점수 입력, 종료는 -1: ");
    scanf("%d", &score);
}
```

### 흐름 시각화

```text
첫 입력
  |
score != -1 ? -- 아니오 --> 종료
  |
 예
  v
처리/출력 -> 다음 입력 ----┘
```

`-1`처럼 데이터가 끝났음을 알리는 특별한 값을 **센티널 값**이라고 부른다. 실제 점수 범위가 `0~100`이므로 `-1`은 종료 값으로 쓰기 좋다.

## 2.2 `do while`: 적어도 한 번은 실행한다

메뉴를 한 번은 보여 준 뒤 종료 여부를 확인하고 싶다면 쓸 수 있다.

```c
int menu;

do {
    printf("1. 입력  0. 종료\n");
    scanf("%d", &menu);
} while (menu != 0);
```

```text
while      : 먼저 물어 보고 들어간다.
do while   : 먼저 한 번 실행하고 물어 본다.
```

## 2.3 `break`: 반복 전체를 끝낸다

```c
for (int i = 1; i <= 5; i++) {
    int score;
    scanf("%d", &score);

    if (score == -1) {
        break;
    }

    printf("%d\n", score);
}
```

두 번째 입력이 `-1`이라면 세 번째부터는 입력하지 않는다.

## 2.4 `continue`: 현재 회차의 나머지만 건너뛴다

```c
if (score < 0 || score > 100) {
    printf("잘못된 점수는 제외합니다.\n");
    continue;
}
sum = sum + score;
valid_count++;
```

잘못된 점수가 들어온 회차는 합계와 유효 개수에 포함하지 않고, 다음 반복을 계속한다.

### `break`와 `continue`의 차이 그림

```text
입력 오류
  |
continue ----> 이번 입력만 제외 ----> 다음 입력

종료값 -1
  |
break -------> 반복문 밖으로 이동 --> 결과 출력
```

## 3교시: 통계를 만드는 네 개의 상자

점수 여러 개에서 통계를 만들 때는 보통 다음 상자를 유지한다.

```text
sum          valid_count         pass_count         max_score/min_score
전체 합계     유효한 입력 개수      합격 점수 개수      현재까지 최고/최저
```

## 3.1 합계와 개수

```c
int sum = 0;
int valid_count = 0;

sum += score;
valid_count++;
```

평균은 반복이 끝난 뒤 계산한다.

```c
double average = (double)sum / valid_count;
```

유효 입력이 하나도 없을 수 있다면 `valid_count > 0`인지 먼저 검사해야 0으로 나누는 오류를 피한다.

## 3.2 합격자 수

```c
if (score >= 60) {
    pass_count++;
}
```

여기서 `score` 하나를 과목 점수로 볼지, 학생의 평균으로 볼지는 요구사항에 따라 달라진다. 오늘 미니 과제에서는 학생당 대표 점수 한 개를 입력받고, 금요일에는 각 점수의 등급까지 출력하도록 확장한다.

## 3.3 최고점과 최저점

최고점은 "지금까지 본 것 중 가장 큰 값"이다.

```text
입력 순서: 72, 88, 65, 95

처음 본 72를 최고점으로 둔다: max = 72
88은 max보다 큼                 max = 88
65는 max보다 작음               max = 88
95는 max보다 큼                 max = 95
```

가장 안전한 방법은 첫 번째 유효 점수가 들어올 때 최고점과 최저점을 함께 설정하는 것이다.

```c
if (valid_count == 0) {
    max_score = score;
    min_score = score;
} else {
    if (score > max_score) {
        max_score = score;
    }
    if (score < min_score) {
        min_score = score;
    }
}

sum += score;
valid_count++;
```

여기서 비교 시점에는 `valid_count`가 아직 증가하기 전이다. 첫 유효 입력인지 판단한 후 누적하는 순서를 추적할 수 있어야 한다.

### 시각화: 입력 `72, 88, 65`의 통계 상자

| 입력 | `sum` | `valid_count` | `pass_count` | `max_score` | `min_score` |
|---:|---:|---:|---:|---:|---:|
| 시작 | 0 | 0 | 0 | 미정 | 미정 |
| 72 | 72 | 1 | 1 | 72 | 72 |
| 88 | 160 | 2 | 2 | 88 | 72 |
| 65 | 225 | 3 | 3 | 88 | 65 |

## 오류 수정 문제: 최저점이 계속 0이다

```c
int min_score = 0;
int score;

for (int i = 1; i <= 3; i++) {
    scanf("%d", &score);
    if (score < min_score) {
        min_score = score;
    }
}
printf("최저점: %d\n", min_score);
```

점수가 `72 88 65`일 때 `min_score`는 왜 `0`에서 바뀌지 않는가?

### 수정 과제

첫 입력을 최저점으로 삼는 방법으로 코드를 고친다. 최고점도 함께 계산하도록 확장한다.

<details>
<summary>해설</summary>

정상 점수는 모두 0 이상이므로 `score < 0`이 한 번도 참이 되지 않는다. 첫 번째 입력일 때 `min_score = score; max_score = score;`로 초기화하고, 이후 입력부터 비교한다.

</details>
