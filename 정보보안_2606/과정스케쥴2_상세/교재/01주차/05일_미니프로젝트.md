## 1교시 A: 프로젝트 전에 푸는 기사형 누적 미니 테스트

실행기를 열기 전에 종이로 푼다. 제한 시간은 20분이다.

### 문제 1: 연산과 형변환

```c
int a = 85;
int b = 90;
int c = 92;
double avg = (a + b + c) / 3;
printf("%.1f", avg);
```

출력과 고쳐야 할 부분을 적는다.

### 문제 2: 증감과 조건

```c
int score = 59;
int count = 0;

if (++score >= 60) {
    count++;
}
printf("%d %d", score, count);
```

### 문제 3: 등급 순서

```c
int avg = 93;
if (avg >= 60) printf("D");
else if (avg >= 70) printf("C");
else if (avg >= 80) printf("B");
else if (avg >= 90) printf("A");
else printf("F");
```

요구사항이 일반적인 A~F 등급이라면 무엇이 잘못되었는가?

### 문제 4: 과락 논리식

```c
int avg = 75;
int korean = 100, english = 85, math = 39;

if (avg >= 60 && korean >= 40 && english >= 40 && math >= 40)
    printf("PASS");
else
    printf("FAIL");
```

### 문제 5: 반복과 누적

```c
int sum = 0;
int pass = 0;

for (int score = 40; score <= 100; score += 20) {
    sum += score;
    if (score >= 60) pass++;
}
printf("%d %d", sum, pass);
```

### 문제 6: 오류 입력 제외

```c
int scores[4] = {80, 120, 60, -5};
int sum = 0;
int count = 0;

for (int i = 0; i < 4; i++) {
    if (scores[i] < 0 || scores[i] > 100) continue;
    sum += scores[i];
    count++;
}
printf("%d %.1f", count, (double)sum / count);
```

배열은 다음 주 본격 학습 대상이다. 오늘은 네 값이 순서대로 선택된다고 보고 추적한다.

### 문제 7: 최고점 초기화

```c
int max_score = 0;
int score = 0;

if (score > max_score) max_score = score;
printf("%d", max_score);
```

점수 범위에서는 결과가 맞아 보인다. 그래도 "첫 입력으로 초기화"하는 규칙이 일반적으로 더 좋은 이유를 한 문장으로 쓴다.

### 문제 8: 메뉴 종료

```c
int menu = 2;
int count = 0;

while (1) {
    count++;
    if (menu == 0) break;
    menu--;
}
printf("%d", count);
```

## 1교시 B: 프로젝트 요구사항 읽기

## 프로젝트 상황

소규모 학습반의 평가가 끝났다. 교사는 학생 수만큼 점수를 입력하고 다음 정보를 바로 확인하려 한다.

- 각 학생의 점수가 올바른 범위인지
- 학생별 등급과 합격 여부
- 반 전체 평균
- 합격한 학생 수
- 최고점과 최저점

지금은 배열을 배우기 전이므로 학생별 결과를 오래 저장해서 다시 정렬하지 않는다. 한 명씩 입력하여 결과를 즉시 출력하고, 전체 통계에 필요한 값만 누적한다.

## 반드시 만족해야 하는 기능 요구사항

### 입력

1. 처리할 학생 수를 입력받는다.
2. 학생 수는 `1~10`만 허용한다.
3. 학생별 점수 한 개를 정수로 입력받는다.
4. 점수는 `0~100`만 허용한다.
5. 잘못된 점수가 입력되면 그 학생 순번의 점수를 다시 입력받아야 하며, 잘못된 값은 통계에 들어가지 않는다.

### 학생별 처리

6. 각 학생의 등급을 출력한다.

| 점수 | 등급 |
|---:|---|
| 90~100 | A |
| 80~89 | B |
| 70~79 | C |
| 60~69 | D |
| 0~59 | F |

7. `60`점 이상은 합격, 미만은 불합격으로 출력한다.

### 전체 통계

8. 모든 학생 입력이 끝나면 합계, 평균, 합격자 수, 불합격자 수, 최고점, 최저점을 출력한다.
9. 평균은 소수 첫째 자리까지 출력한다.
10. 합격률을 소수 첫째 자리까지 출력한다.

### 사용해야 하는 문법

- `int`, `double`
- `scanf`, `printf`
- 산술 연산과 형변환
- `if / else if / else`
- `&&` 또는 `||`
- `for` 또는 `while`
- 누적 변수
- 최고/최저 갱신

## 기능을 처리 순서로 번역하기

```text
   |

   |

   |

   |
합계, 합격 수, 최고/최저 __________________
   |
반복 종료
   |
```

### 설계 확인 질문

1. 불합격자 수를 별도 상자에 누적해야만 하는가, 아니면 마지막에 계산할 수 있는가?
2. 최고점/최저점은 언제 처음 설정하는 것이 안전한가?
3. 잘못 입력된 점수를 받은 반복 회차를 다음 학생으로 넘어가면 무엇이 잘못되는가?
4. 학생 수가 0이면 평균 계산에서 어떤 문제가 생기는가?

<details>

4. 0으로 나누게 되므로 학생 수를 먼저 `1~10`으로 제한한다.



```c
while (/* 올바르지 않은가? */) {
```

| 등급 `if / else if` | 조건 순서를 바꾸면 어떤 점수가 틀리는가? |
| 최고/최저 갱신 | 첫 학생을 별도로 처리하는 이유는 무엇인가? |
| 평균/합격률 | 형변환이 없으면 어떤 테스트에서 오류가 보이는가? |

## 제출 직전 제한 시간 도전

최종 파일이 이미 작동한 뒤 실시한다. 기존 파일을 보지 않고 새 종이에 아래 함수 없는 의사코드를 10분 안에 작성한다.

```text
입력받아야 할 값:
초기화할 통계 값:
반복 시작 조건:
오류 입력 처리:
한 학생 처리 뒤 갱신할 통계:
반복 종료 뒤 출력할 결과:
```

이 활동은 코드를 한 번 완성했다고 해서 이해했다고 착각하지 않게 해 준다. 머릿속에서 흐름을 다시 꺼낼 수 있으면 다음 주 배열과 함수 학습으로 넘어갈 기반이 생긴다.

## 선택 확장 과제

필수 기능과 테스트를 모두 마친 학생만 선택한다.

### 확장 A: 메뉴 방식

다음 메뉴를 추가한다.

```text
1. 새 학급 입력 시작
2. 직전 결과 다시 보기
0. 종료
```

주의: 배열을 사용하지 않으므로 학생별 점수 목록을 다시 출력하는 기능은 요구하지 않는다. 직전 통계 값만 유지한다.

### 확장 B: 장학생 판정

- 학생별 점수가 `95` 이상이면 `장학생 후보`를 함께 출력한다.
- 전체 장학생 후보 수를 통계에 추가한다.
- `94`, `95` 경계값 테스트를 반드시 추가한다.

### 확장 C: 합격률에 따른 반 메시지

| 합격률 | 메시지 |
|---:|---|
| 80% 이상 | `우수 반` |
| 50% 이상 80% 미만 | `보통 반` |
| 50% 미만 | `보충 학습 필요` |

조건문 순서와 소수 비교에 유의한다.
