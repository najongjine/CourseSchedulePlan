# 1주차 학습교재: C 제어문과 연산 추적

## 교재의 목적

이 교재는 C 문법을 외워 맞히는 수업이 아니다. 정보처리기사 실기에서 마주칠 수 있는 짧고 복합적인 코드를 학생이 손으로 추적하고, 고치고, 결국 스스로 작성하는 것을 목표로 한다.

1주차의 최종 과제는 **점수 판정 및 통계 프로그램**이다. 월요일부터 목요일까지 완성 코드 한 벌을 외우지 않는다. 대신 값 저장, 계산, 조건 판정, 반복 누적, 최댓값/최솟값, 입력 검증과 메뉴 흐름을 순서대로 훈련한다. 금요일에는 요구사항과 테스트 데이터만 보고 빈 파일에서 구현한다.

## 수강생에게 먼저 할 말

프로그래밍은 영어를 잘하거나 수학을 빠르게 푸는 사람만의 일이 아니다. 처음에는 아래 세 가지 행동을 정확히 하는 것이 더 중요하다.

1. 코드를 위에서 아래로 한 줄씩 읽는다.
2. 값이 바뀔 때마다 종이에 변수 상자를 갱신한다.
3. 예상 결과를 먼저 적은 뒤 컴파일하고 실행하여 확인한다.

틀린 예측은 실패가 아니라 가장 중요한 학습 기록이다. "왜 내가 7이라고 생각했는데 실제 출력은 8이었는가"를 찾는 과정이 시험의 코드 추적 능력이 된다.

## 1주차 전체 지도

| 일차 | 핵심 주제 | 이미 배운 것을 함께 쓰는 지점 | 당일 산출물 |
|---|---|---|---|
| 1일 | 변수, 자료형, 연산, 형변환, 증감 | 첫날이므로 입력-계산-출력 흐름 확립 | `day1_score.c`, 추적표 |
| 2일 | 조건문, 논리식, 경계값 | 입력, 산술 평균, 형변환을 조건과 결합 | `day2_judge.c`, 경계값 테스트표 |
| 3일 | 반복문, 누적, 개수, 최고/최저 | 조건 판정과 평균 계산을 여러 학생에 반복 | `day3_summary.c`, 반복 추적표 |
| 4일 | 입력 검증, 센티널, 메뉴 반복 | 조건+반복+누적+최고/최저를 메뉴 프로그램에 결합 | `day4_menu.c`, 프로젝트 설계서 |
| 5일 | 독립 구현 및 검증 | 1~4일의 모든 문법 | `score_report.c`, 테스트/설명 기록 |

## 매일 사용하는 학습 루틴

```text
읽기          손 추적             직접 입력           변경/구현
코드의 역할 -> 변수표로 결과 예측 -> 컴파일과 실행 확인 -> 조건을 바꿔 재작성
```

### 손 추적 표 작성 규칙

| 실행 줄 | 입력/조건 | `score` | `sum` | `count` | 출력 |
|---|---:|---:|---:|---:|---|
| 초기값 | - | - | 0 | 0 | - |
| 입력 후 | 85 | 85 | 0 | 0 | - |
| 누적 후 | - | 85 | 85 | 1 | - |

- 값이 바뀌지 않는 줄은 억지로 모두 쓰지 않아도 된다.
- 조건식은 `참` 또는 `거짓`을 반드시 적는다.
- 출력은 실행 시점의 값을 적는다. 나중 값으로 고쳐 쓰지 않는다.

## 파일 작성 규칙

- 소스 파일 인코딩: UTF-8
- 파일명: 교재에 지정된 영문 파일명 사용
- 컴파일 예시: `gcc day1_score.c -o day1_score`
- 실행 예시: Windows PowerShell에서는 `.\day1_score.exe`
- 제출물에는 소스 코드, 테스트 입력/예상 출력/실제 출력, 오류 수정 기록을 함께 둔다.
