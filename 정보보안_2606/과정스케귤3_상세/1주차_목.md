# 1주차 목요일: random, 힌트, 난이도로 숫자 게임 조립하기

> 주간 위치: Python 첫 감각과 콘솔 숫자 게임 4/5  
> 오늘의 핵심 질문: 랜덤 정답, 힌트, 횟수 제한을 합치면 게임 흐름이 되는가?  
> 당일 산출물: 업다운 숫자 맞히기 1차 버전, 난이도 선택 초안

## 오늘 목표

- `random` 모듈로 임의의 숫자를 만든다.
- 사용자의 입력값과 정답을 비교해 업/다운 힌트를 출력한다.
- 반복문과 조건문을 결합해 게임 흐름을 만든다.
- 난이도 선택, 시도 횟수 제한, 재시작의 설계 뼈대를 잡는다.

## 1~2교시 이론 연결 포인트

| 이론 키워드 | 프로그래밍 연결 |
|---|---|
| 난수 | 매번 달라지는 값으로 테스트 흐름이 달라진다. |
| 조건 분기 | 정답보다 큰지 작은지에 따라 힌트가 갈린다. |
| 반복 제어 | 맞히거나 기회를 다 쓰면 반복을 멈춘다. |
| 테스트 케이스 | 랜덤을 잠시 고정하고 예상 결과를 검증한다. |

## 3~6교시 운영 흐름

| 교시 | 활동 | 끝났을 때 할 수 있어야 하는 것 |
|---|---|---|
| 3교시 | `random.randint()`와 랜덤 정답 만들기 | 실행할 때마다 다른 정답을 생성한다. |
| 4교시 | 업/다운 힌트 구현 | 입력값과 정답을 비교해 다른 메시지를 출력한다. |
| 5교시 | 시도 횟수 제한과 성공/실패 판정 | 맞히거나 횟수를 다 쓰면 게임을 끝낸다. |
| 6교시 | 난이도 선택 설계와 1차 통합 | 쉬움/보통/어려움에 따라 범위와 기회를 바꾼다. |

## 핵심 개념

- `import random`
- `random.randint()`
- 랜덤 정답
- 업/다운 힌트
- 성공 플래그
- 시도 횟수 제한
- 난이도 선택
- 테스트를 위한 정답 고정

## 수업 전개 뼈대

### 1. 랜덤 운세 뽑기에서 시작

- 숫자 게임 전에 랜덤으로 문장 하나를 뽑는 감각을 만든다.
- `random.randint(1, 3)` 결과에 따라 다른 운세를 출력한다.
- 랜덤도 결국 숫자를 하나 만드는 코드라는 점을 확인한다.

### 2. 랜덤 정답 만들기

- `answer = random.randint(1, 10)`으로 정답을 만든다.
- 개발 중에는 정답을 잠깐 출력해서 테스트한다.
- 테스트가 끝나면 정답 출력 코드를 지운다.

### 3. 업다운 힌트 구현

- 입력값이 정답보다 작으면 "더 큰 수", 크면 "더 작은 수"를 출력한다.
- 같으면 성공 메시지를 출력한다.
- `if / elif / else` 순서와 조건식을 손으로 추적한다.

### 4. 게임 1차 버전 조립

- 최대 5번까지 입력받도록 `while`을 사용한다.
- 맞히면 `break`로 반복을 끝낸다.
- 끝난 뒤 성공/실패 메시지를 구분한다.

### 5. 난이도 선택 초안

- 쉬움: 1~10, 5번
- 보통: 1~50, 7번
- 어려움: 1~100, 10번
- 선택값에 따라 `max_number`, `max_try`를 바꾸는 구조를 만든다.
