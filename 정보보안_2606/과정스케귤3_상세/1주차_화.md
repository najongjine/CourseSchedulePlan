# 1주차 화요일: 자료형, 형변환, 조건문으로 판정 만들기

> 주간 위치: Python 첫 감각과 콘솔 숫자 게임 2/5  
> 오늘의 핵심 질문: 입력값에 따라 프로그램이 다른 길로 가게 하려면 무엇이 필요한가?  
> 당일 산출물: 성적 합격/불합격 판정기, 나이대 판정기, 간단 메뉴 판정 프로그램

## 오늘 목표

- 문자열과 숫자 자료형의 차이를 이해한다.
- `int()` 형변환을 사용해 입력값을 계산 가능한 숫자로 바꾼다.
- `if`, `elif`, `else`로 조건에 따라 다른 결과를 출력한다.
- 비교 연산자와 논리 연산자를 정보처리기사 코드 추적 문제와 연결한다.

## 1~2교시 이론 연결 포인트

| 이론 키워드 | 프로그래밍 연결 |
|---|---|
| 자료형 | 문자열과 정수는 다르게 처리된다. |
| 비교 연산자 | `>=`, `<`, `==` 결과는 참 또는 거짓이다. |
| 조건문 | 조건이 참일 때만 특정 문장을 실행한다. |
| 복합 조건 | `and`, `or`로 조건을 묶는다. |

## 3~6교시 운영 흐름

| 교시 | 활동 | 끝났을 때 할 수 있어야 하는 것 |
|---|---|---|
| 3교시 | 자료형, 형변환, 비교 연산 | 입력값을 숫자로 바꾸고 비교한다. |
| 4교시 | `if / else` 판정 | 점수에 따라 합격/불합격을 출력한다. |
| 5교시 | `elif`, 복합 조건 | 여러 구간을 나누어 등급이나 상태를 판정한다. |
| 6교시 | 미니 판정기 제작, 경계값 테스트 | 59, 60, 89, 90 같은 경계값을 직접 확인한다. |

## 수업 전개 뼈대

### 1. 문자열 숫자와 진짜 숫자 비교

- `"10" + "20"`과 `10 + 20`의 차이를 확인한다.
- `input()` 결과의 자료형을 `type()`으로 확인한다.
- 형변환이 필요한 상황을 계산기 예제로 다시 연결한다.

### 2. 성적 합격/불합격 판정기

- 점수를 입력받아 60점 이상이면 합격, 아니면 불합격을 출력한다.
- 60점을 기준으로 59, 60, 61을 테스트한다.
- 조건식의 참/거짓을 손으로 먼저 예측하고 실행한다.

### 3. 등급 판정기

- 90점 이상 A, 80점 이상 B, 70점 이상 C, 그 외 보충으로 나눈다.
- `if` 순서가 바뀌면 결과가 달라지는 사례를 보여준다.
- 위에서 아래로 조건을 검사한다는 점을 변수 추적표와 연결한다.

### 4. 간단 메뉴 판정 프로그램

- `1. 인사`, `2. 계산`, `3. 종료` 같은 메뉴 번호를 입력받는다.
- 선택값에 따라 다른 문장을 출력한다.
- 금요일 숫자 게임의 메뉴와 난이도 선택으로 이어질 준비를 한다.
