9주차: 예측의 시작, Linear Regression의 한계와 LightGBM

주차 목표
- AI가 마법이 아니라 데이터로 규칙을 추정하는 프로그램이라는 감각을 만든다.
- Linear Regression으로 예측의 가장 단순한 출발점을 경험한다.
- Linear Regression이 잘 맞는 문제와 잘 안 맞는 문제를 비교한다.
- LightGBM이 회귀와 분류를 모두 처리할 수 있다는 것을 경험한다.

핵심 내용
- 데이터셋
- feature
- label
- train/test split
- 회귀
- 분류
- Linear Regression
- 직선으로 예측한다는 의미
- 선형 모델의 한계
- LightGBM Regressor
- LightGBM Classifier
- 정확도
- 오차
- 과적합 맛보기

정보처리기사 연결
- AI 기초
- 데이터 전처리
- 모델 평가
- 통계 기초
- 데이터 품질
- 회귀와 분류의 차이
- 입력 데이터와 출력 데이터 구분

미니 주제
- 공부 시간으로 시험 점수 예측: Linear Regression
- 기온으로 전기 사용량 예측: 직선 모델의 한계 관찰
- LightGBM Regressor로 같은 예측 다시 해보기
- 붓꽃 또는 간단 건강 데이터 분류: LightGBM Classifier
- 회귀 결과와 분류 결과의 출력 형태 비교

주차 프로젝트
- 예측 모델 비교 실험실
- Linear Regression으로 먼저 예측기를 만들고, 예측이 잘 되는 데이터와 잘 안 되는 데이터를 비교한다.
- 같은 데이터를 LightGBM Regressor로 다시 학습해 차이를 확인한다.
- 별도 분류 데이터로 LightGBM Classifier를 사용해 "A인가 B인가"를 예측한다.
- 최종 산출물은 "회귀/분류 차이, Linear Regression의 장점과 한계, LightGBM이 필요한 상황"을 정리한 실행 코드와 짧은 보고서로 한다.

운영 메모
- 초반 4주는 절대 문법만 나열하지 않는다. 매주 "작게라도 돌아가는 결과물"을 남긴다.
- 정보처리기사 파트와 프로그래밍 파트는 같은 단어를 반복해서 연결한다. 예를 들어 배열을 배운 주에는 Python list와 React 배열 렌더링으로, SQL을 배운 주에는 Postgres 게시판 DB로, API를 배운 주에는 FastAPI 요청/응답으로, AI 기초를 배운 주에는 실제 예측기로 연결한다.
- 비전공자 대상이므로 수학 설명은 최소화하고, 입력값이 바뀌면 결과가 어떻게 바뀌는지 눈으로 확인하게 한다.
- 프로젝트는 완성보다 "요구사항을 기능으로 쪼개고, 오류를 고치고, 자기 코드로 설명하는 경험"을 더 중요하게 본다.
- 마지막 주 결과물은 포트폴리오로 보여줄 수 있도록 발표 시간을 반드시 둔다.
