## 최종 프로젝트 요구사항

### 필수 기능

- 게임 시작 메시지를 출력한다.
- 난이도를 선택한다.
- 난이도에 따라 숫자 범위와 시도 횟수가 달라진다.
- 랜덤 정답을 만든다.
- 사용자가 숫자를 입력한다.
- 입력값이 정답보다 작으면 업 힌트를 준다.
- 입력값이 정답보다 크면 다운 힌트를 준다.
- 정답을 맞히면 성공 메시지와 시도 횟수를 출력한다.
- 기회를 모두 쓰면 실패 메시지와 정답을 출력한다.

### 선택 기능

- 게임 재시작
- 잘못된 메뉴 입력 처리
- 숫자가 아닌 입력에 대한 안내
- 남은 기회 표시
- 범위를 벗어난 숫자 입력 안내
- 최고 기록 또는 최소 시도 횟수 기록

## 수업 전개 뼈대

### 1. 요구사항을 기능으로 쪼개기

- "숫자 맞히기 게임"을 한 번에 만들려고 하지 않는다.
- 아래 순서로 작은 기능을 체크한다.

| 순서 | 기능 | 확인 방법 |
|---:|---|---|
| 1 | 시작 메시지 | 실행하면 안내 문장이 보인다. |
| 2 | 난이도 선택 | 선택값에 따라 범위와 횟수가 출력된다. |
| 3 | 랜덤 정답 | 정해진 범위 안에서 정답이 생성된다. |
| 4 | 입력과 비교 | 업/다운/정답 메시지가 갈린다. |
| 5 | 반복과 종료 | 맞히거나 기회가 끝나면 멈춘다. |
| 6 | 재시작 | 다시 할지 묻고 새 게임을 시작한다. |

### 2. 기본 버전 먼저 완성

- 정답 범위는 처음에 1~10으로 고정한다.
- 시도 횟수도 5번으로 고정한다.
- 기본 게임이 돌아간 뒤 난이도와 재시작을 붙인다.

### 3. 난이도 붙이기

- 메뉴 입력값에 따라 `max_number`, `max_try`를 정한다.
- 잘못된 메뉴는 기본 난이도로 돌리거나 다시 입력받는다.
- 난이도를 바꿔도 나머지 게임 코드는 최대한 그대로 쓰게 한다.

### 4. 테스트와 오류 수정

- 정답을 잠깐 7로 고정하고 입력별 결과를 확인한다.
- 정답보다 작은 값, 큰 값, 같은 값, 마지막 기회 정답, 전부 실패를 테스트한다.
- 입력 실수 상황은 선택 기능으로 처리한다.

### 5. 코드 설명과 발표

- 변수 3개 이상을 골라 역할을 설명한다.
- 반복문이 언제 끝나는지 설명한다.
- 내가 고친 오류 1개를 말한다.

## 테스트 케이스

| 상황 | 예시 입력 | 기대 결과 |
|---|---|---|
| 정답보다 작음 | 정답 7, 입력 3 | 더 큰 수 안내 |
| 정답보다 큼 | 정답 7, 입력 9 | 더 작은 수 안내 |
| 정답 맞힘 | 정답 7, 입력 7 | 성공 및 시도 횟수 출력 |
| 기회 소진 | 5번 모두 오답 | 실패 및 정답 출력 |
| 난이도 선택 | 쉬움 선택 | 1~10 범위와 5번 기회 |
| 재시작 | `y` 입력 | 새 정답으로 다시 시작 |
