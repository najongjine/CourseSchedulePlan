- 산출물: 데이터를 불러오고 필요한 열을 선택하는 코드

5주차 3일(수) - scikit-learn 모델 가져다 써 보기
- 이론: 선형회귀는 "숫자가 늘 때 결과도 어느 정도 함께 늘거나 줄어드는 문제"에 써 볼 수 있는 도구 / 내부 계산을 모두 만들지 않아도 모듈을 가져다 사용할 수 있음
- 실습: `LinearRegression`을 불러와 `fit()`으로 준비된 데이터를 넣고, `predict()`로 공부 시간이 다른 학생의 예상 점수를 출력하기
- 프로젝트 연결: 수강생이 입력한 공부 시간에 대해 예상 점수를 보여주는 짧은 프로그램 만들기
- 산출물: scikit-learn을 사용한 공부 시간 기반 점수 예측 프로그램

5주차 4일(목) - 선형회귀로 잘 되는 것과 안 되는 것
- 이론: 공부 시간이 많아질수록 점수가 달라지는 예측에는 써 볼 수 있지만, 좋아하는 색이나 반 번호처럼 관계를 기대하기 어려운 값으로는 잘 예측되지 않음
- 실습: `공부 시간`으로 예측한 결과와 `좌석 번호`로 예측한 결과를 실행해 보고, 몇 명의 실제 점수와 예상 점수를 눈으로 비교하기
- 프로젝트 연결: 입력 열을 바꿨을 때 결과가 믿을 만해 보이는지 직접 판단하여 메모하기
- 산출물: 잘 되는 입력과 잘 안 되는 입력을 비교한 실행 결과 및 짧은 소감

5주차 5일(금) - 변수를 많이 넣으면 무조건 좋아질까
- 이론: 모델에 넣는 정보가 모두 도움이 되는 것은 아니며, 필요 없는 정보를 빼는 것이 결과를 더 납득 가능하게 만들 수 있음
- 실습: `공부 시간`, `공부 시간+수면 시간`, `공부 시간+수면 시간+출석 횟수`, `모든 열(좌석 번호 포함)`의 네 가지 조합으로 같은 학생들의 예상 점수를 출력해 비교하기
- 프로젝트 연결: 본인이 가장 납득되는 열 조합을 선택해 입력값을 받는 점수 예측 프로그램 최종본 만들기
- 산출물: scikit-learn 선형회귀 프로젝트 최종본과 "넣을 변수/뺄 변수/그 이유" 기록


6주차: 직선으로 안 되는 데이터와 LightGBM 사용하기
주차 목표: 선형회귀가 잘 맞지 않는 굽은 패턴을 직접 확인하고, 의사결정 나무와 LightGBM이 조건을 나누어 이런 데이터를 다룰 수 있다는 점을 경험한다.

6주차 1일(월) - 직선으로는 곤란한 데이터 맛보기
- 이론: 5주차의 선형회귀는 한 방향으로 늘거나 줄어드는 데이터에는 어울리지만, 내려갔다 다시 올라가는 모양에는 어울리지 않음
- 실습: `기온에 따른 전기 사용량` 데이터 보기(추운 날과 더운 날 모두 사용량이 높고, 선선한 날에는 낮음), 5주차 방식으로 `LinearRegression` 예측 실행하기
- 프로젝트 연결: 실제 값과 선형회귀 예측을 표로 보며 "직선 하나로는 이 모양을 따라가기 어렵다" 확인하기
- 산출물: 선형회귀가 굽은 데이터에서 어색한 결과를 내는 실행 코드와 소감

6주차 2일(화) - 의사결정 나무는 if문을 스스로 만든다
- 이론: 사람이 쓴다면 `기온이 10도보다 낮으면`, `28도보다 높으면`처럼 조건문으로 나눌 수 있음 / 의사결정 나무는 이런 `if` 기준을 데이터를 보며 스스로 찾는 모델
- 실습: 종이에 간단한 온도 조건문을 써서 전기 사용량 구간을 예측한 뒤, `DecisionTreeRegressor`를 가져와 같은 데이터를 학습시키고 결과 보기
- 프로젝트 연결: 직선 대신 여러 조건으로 나누면 굽은 데이터도 따라갈 수 있음을 확인하기
- 산출물: 사람이 쓴 조건문 메모와 의사결정 나무 실행 예제

6주차 3일(수) - LightGBM은 나무를 똑똑하게 모아 쓴다
- 이론: 나무 하나보다 여러 나무의 판단을 합치면 더 복잡한 패턴을 다룰 수 있음 / LightGBM은 표 데이터에서 회귀와 분류 모두에 사용할 수 있는 강력한 도구
- 실습: `LGBMRegressor`를 불러와 기온-전기 사용량 데이터를 학습하고, 선형회귀 결과와 나란히 출력하기
- 프로젝트 연결: 입력한 기온에 따라 예상 전기 사용량을 보여주는 짧은 프로그램 만들기
- 산출물: LightGBM 회귀 예측 프로그램 첫 버전

6주차 4일(목) - 분류도 되는 LightGBM
- 이론: 숫자를 예상하는 회귀뿐 아니라 "A인가 B인가"를 고르는 분류에도 같은 도구를 사용할 수 있음 / 모델 사용 코드는 비슷하고 질문이 달라짐
- 실습: 화면에 표시된 `x, y 좌표의 구불구불한 빨강/파랑 구역` 준비 데이터를 이용하여 `LGBMClassifier`를 실행하고, 새 좌표가 어느 색인지 예측하기
- 프로젝트 연결: 회귀 예제와 분류 예제에서 바뀐 부분이 모델 이름과 정답 형태라는 점 찾아보기
- 산출물: LightGBM 회귀/분류 예제 두 개와 차이 정리

6주차 5일(금) - 강력하지만 처음 보는 범위에는 약하다
- 이론: LightGBM은 배운 데이터 안의 구불구불한 패턴은 잘 찾지만, 한 번도 보지 못한 범위 밖 값의 흐름을 알아서 이어 그리는 모델은 아님
- 실습: 학습 데이터가 `-5도~35도`일 때 그 안의 기온과 `45도` 같은 범위 밖 기온을 넣어 결과를 비교하고, 왜 결과를 그대로 믿으면 안 되는지 이야기하기
- 프로젝트 연결: 기온을 입력받아 LightGBM 예측값을 출력하되, 학습 범위를 벗어나면 "처음 보는 범위라 예측을 믿기 어렵습니다" 안내 추가하기
- 산출물: LightGBM 프로젝트 최종본과 "잘하는 것 / 조심할 것" 두 문장 기록


7주차: Dense 레이어를 바꾸어 놀아 보는 딥러닝 체험
주차 목표: 교사가 준비한 실행 코드에서 Dense 레이어와 activation function을 자유롭게 바꾸어 보며, 수치 회귀와 수치 분류가 되는 모습을 체험한다. 또한 단순 표 데이터에서는 LightGBM이 더 편한데도 딥러닝을 배우는 이유가 이미지와 문장 같은 비정형 데이터에 있음을 이해한다.

7주차 1일(월) - LightGBM도 되는데 딥러닝은 왜 볼까
- 이론: 공부 시간, 기온, 좌표처럼 열이 이미 정리된 숫자 데이터라면 LightGBM이 매우 편하고 강력함 / 딥러닝의 진짜 장점은 사진 속 고양이와 호랑이, 개와 늑대 또는 문장 속 인사와 욕설처럼 사람이 규칙 열을 만들기 어려운 데이터에서 커짐
- 실습: 표 데이터의 열은 바로 적을 수 있지만, 사진을 보고 "고양이인지 호랑이인지 판단하는 열"을 직접 정하는 것은 어렵다는 점 이야기하기
- 프로젝트 연결: 다음 주 이미지 수업에 앞서, 이번 주에는 다루기 쉬운 숫자 데이터로 Dense 레이어를 조립해 보는 연습만 한다고 안내
- 산출물: "표 데이터면 LightGBM / 이미지·문장이면 딥러닝을 배울 이유가 있음" 한 줄 정리

7주차 2일(화) - Dense 레이어와 activation 버튼 바꾸듯 써 보기
- 이론: Dense 레이어는 숫자 입력을 다음 숫자 묶음으로 바꾸는 블록 / activation은 블록 사이에 넣어 단순한 직선만이 아닌 모양도 표현하게 돕는 선택지 / 내부 수학 계산은 이번 과정의 목표가 아님
- 쉬운 설명 소개: 신경망은 사람이 `if` 규칙을 읽을 수 있게 적어 주는 방식이 아니다. 숫자들을 계속 더하고 곱하고, activation으로 모양을 꺾고 눌러 보며, 학습 과정에서 내부 값들을 스스로 조정하다가 결과를 맞히게 되는 별종 같은 도구이다. 어떤 숫자들이 모여 왜 정확히 "고양이답다" 또는 "이 문장답다"고 판단했는지를 사람이 100% 속 시원하게 설명하기는 어렵다. 이번 수업에서는 내부를 완벽히 이해하려고 붙잡히지 말고, 블록을 바꾸면 결과가 달라진다는 사용 감각을 먼저 익힌다.
- 실습: 교사가 제공한 실행 틀에서 `Linear(1, 8)`, `Linear(8, 1)`처럼 층 크기를 바꾸고, `ReLU`, `Tanh`, `Sigmoid` 중 하나를 골라 실행하기
- 프로젝트 연결: 코드를 새로 짜는 대신 바꾼 레이어 목록과 실행 결과 그림을 저장하는 방법 익히기
- 산출물: 레이어/activation 조합 3개의 실행 결과 캡처 또는 기록, "신경망을 이번 과정에서 어디까지 이해하면 되는가" 한 문장 정리

7주차 3일(수) - Dense 회귀 체험: 기온으로 전기 사용량 예측하기
- 이론: 회귀는 결과가 숫자인 문제 / 6주차와 같은 굽은 데이터도 Dense 블록 조합에 따라 따라가는 모양이 달라질 수 있음
- 실습: 준비된 `기온-전기 사용량` 실행 코드에서 Dense 레이어 수와 activation만 바꾸어 예측 선 모양 비교하기
- 프로젝트 연결: 결과가 가장 자연스러워 보이는 조합 하나를 선택하고 LightGBM 결과와 눈으로 비교하기
- 산출물: Dense 회귀 예제와 "LightGBM 쪽이 더 간단했는지" 소감

7주차 4일(목) - Dense 분류 체험: 원 안쪽과 바깥쪽 구분하기
- 이론: 분류는 결과가 종류인 문제 / 직선 하나로 가르기 힘든 둥근 구역도 여러 Dense 블록과 activation을 넣으면 모양을 표현할 수 있음
- 실습: `x, y 좌표가 원 안이면 파랑, 바깥이면 빨강`인 준비 데이터를 실행하고, 층 개수와 activation을 바꾸어 결과 색칠 화면 비교하기
- 프로젝트 연결: 수강생 각자 가장 잘 구분되어 보이는 레이어 조합을 고르고 이유 말하기
- 산출물: Dense 분류 예제와 선택한 조합 기록

7주차 5일(금) - 숫자 장난감에서 이미지와 문장으로 넘어가기
- 이론: 이번 주 숫자 데이터는 원리를 맛보기 위한 장난감이며, 실제 목적은 픽셀 수천 개가 있는 이미지나 단어가 이어지는 문장처럼 사람이 특징을 일일이 정하기 어려운 것을 다루는 것 / 다음 주 CNN은 이미지에 맞게 만든 딥러닝 블록
- 실습: 회귀와 분류 중 하나를 골라 레이어와 activation 조합을 자유롭게 바꾼 결과를 짧게 시연하기
- 프로젝트 연결: "고양이/호랑이/개/늑대 사진 분류" 또는 "좋은 말/나쁜 말 문장 분류"처럼 나중에 만들어 보고 싶은 비정형 데이터 앱 아이디어 작성
- 산출물: Dense 체험 프로젝트 최종본과 다음 이미지 수업으로 이어지는 앱 아이디어 1개


8주차: 이미지 데이터와 CNN, MNIST 숫자 인식
주차 목표: 이미지가 픽셀 데이터라는 사실과 CNN이 주변 무늬를 찾는 방식을 이해하고, 손글씨 숫자 분류기를 학습한다.

8주차 1일(월) - 컴퓨터가 보는 이미지
- 이론: 픽셀, 흑백/컬러 채널, 해상도 / 이미지가 표 데이터보다 복잡한 이유
- 실습: MNIST 이미지 몇 장을 화면에 표시하고 픽셀 값과 정답 확인
- 프로젝트 연결: "손글씨 숫자를 맞히는 프로그램"의 입력과 출력 정의
- 산출물: MNIST 탐색 코드와 관찰 메모

8주차 2일(화) - CNN은 무엇을 찾는가
- 이론: 필터, 합성곱, 특징 맵, pooling을 선/모서리 찾기 비유로 이해하기
- 실습: 간단한 흑백 격자에 필터를 적용한 결과 관찰, CNN 층을 통과한 텐서 크기 확인
- 프로젝트 연결: 사용할 작은 CNN 구조 그리기
- 산출물: CNN 구조도와 shape 확인 코드

8주차 3일(수) - MNIST CNN 학습
- 이론: 학습 배치, epoch, 학습/테스트 데이터 / 이미지 분류 loss
- 실습: PyTorch DataLoader와 간단한 CNN으로 학습 실행, 정확도 확인

8주차 4일(목) - 모델은 무엇을 틀리는가
- 이론: 정확도와 오분류 이미지 / 확신이 높아도 틀릴 수 있음 / 직접 만든 모델의 한계
- 실습: 잘못 맞힌 숫자 이미지 표시, 숫자가 흐리거나 기울어진 경우 결과 관찰
- 프로젝트 연결: 예측 결과와 확률을 보여주는 테스트 기능 추가
