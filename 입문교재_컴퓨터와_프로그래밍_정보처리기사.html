이 교재의 목표

이 교재는 프로그래밍을 처음 배우는 사람이 “왜 이런 것을 배우는지”부터 이해하도록 돕기 위해 작성되었습니다. 문법을 외우기 전에 컴퓨터, 언어, 개발 과정의 큰 그림을 먼저 잡는 것이 목표입니다.

핵심 관점

프로그래밍은 코드를 따라 치는 기술만이 아닙니다. 사람이 해결하려는 문제를 컴퓨터가 실행할 수 있는 절차로 바꾸는 사고방식입니다.

왜 배우는가 컴퓨터가 어떤 방식으로 명령을 실행하는지 알면 문법이 덜 낯설어집니다.
무엇을 배우는가 컴퓨터 구조, 프로그래밍 언어, 개발 실무, 정보처리기사의 연결을 배웁니다.
어떻게 써먹는가 작은 프로그램과 프로젝트를 만들 때 필요한 판단 기준을 얻습니다.

1장. 컴퓨터는 왜 만들어졌을까?

컴퓨터는 원래 “계산하는 사람”이었다

오늘날 컴퓨터라고 하면 노트북, 데스크톱, 스마트폰, 서버를 떠올립니다. 하지만 원래 computer라는 말은 기계가 아니라 “계산하는 사람”을 뜻했습니다. 천문, 항해, 세금, 군사, 통계 계산을 사람이 직접 하던 시대에는 계산량이 많고 실수가 잦았습니다.

그래서 빠르고 정확하게 계산하는 기계가 필요해졌습니다. 컴퓨터는 처음부터 유튜브나 게임을 위해 만들어진 것이 아니라, 대량의 계산을 빠르게 처리하기 위해 등장했습니다.

시각화: 계산 도구에서 범용 컴퓨터로

계산기와 컴퓨터의 차이

계산기는 정해진 계산을 빠르게 해줍니다. 그러나 컴퓨터는 명령을 저장해두고 조건 판단, 반복 작업, 데이터 저장, 통신, 화면 출력까지 수행합니다. 

사용자가 로그인했는가?
로그인했다면 메인 화면을 보여준다.
로그인하지 않았다면 로그인 화면으로 보낸다.
장바구니에 상품이 있는가?
있다면 결제 금액을 계산한다.

컴퓨터는 “명령을 저장해두고 순서대로 실행할 수 있는 범용 기계”입니다.

컴퓨터의 핵심 능력

계산복잡한 숫자 계산을 빠르게 수행합니다.
반복지치지 않고 같은 작업을 반복합니다.
저장많은 데이터를 저장하고 다시 꺼냅니다.
판단정해진 조건에 따라 다른 동작을 합니다.

2장. 폰 노이만 모델은 왜 등장했을까?

초기 컴퓨터의 문제

초기 컴퓨터는 작업을 바꾸려면 배선을 바꾸거나 스위치를 다시 설정해야 했습니다. 요리를 바꿀 때마다 주방 구조를 뜯어고쳐야 하는 식당과 비슷했습니다.

저장 프로그램 방식

프로그램도 데이터처럼 메모리에 저장하자는 아이디어입니다. 이 덕분에 컴퓨터는 하드웨어를 매번 바꾸지 않고 프로그램만 바꿔 여러 작업을 수행할 수 있게 되었습니다.

시각화: 폰 노이만 구조

CPU의 기본 동작

CPU는 보통 세 단계로 명령을 처리합니다.

1. Fetch메모리에서 명령어를 가져옵니다.
2. Decode명령어가 무슨 뜻인지 해석합니다.
3. Execute해석한 명령어를 실행합니다.
total = 10 + 20
print(total)

이 두 줄의 코드도 컴퓨터 내부에서는 값을 준비하고, 더하고, 저장하고, 출력 기능으로 보내는 작은 명령들의 묶음입니다.

왜 아직도 폰 노이만 구조를 쓸까?

폰 노이만 구조는 범용성이 좋고, 구조가 비교적 단순하며, 운영체제와 프로그래밍 언어와 하드웨어 산업이 이 틀을 기반으로 발전했기 때문에 여전히 중요합니다.

폰 노이만 병목

CPU가 아무리 빨라도 메모리에서 데이터를 가져오는 속도가 느리면 전체 성능이 떨어집니다. 현대 컴퓨터는 캐시, 멀티코어, GPU, 병렬 처리 등으로 이 문제를 줄입니다.

3장. 프로그래밍 언어는 왜 필요할까?

컴퓨터가 직접 이해하는 언어

컴퓨터는 최종적으로 0과 1로 표현되는 기계어를 실행합니다. 사람이 이진수만 보고 프로그램을 작성하기는 매우 어렵습니다. 그래서 사람이 읽기 쉬운 방식으로 명령을 표현하는 프로그래밍 언어가 등장했습니다. 

10110000 01100001

시각화: 사람이 읽는 코드가 실행되는 과정

왜 자연어로 명령하지 않을까?

사람의 말은 편하지만 애매합니다. “적당히 큰 숫자를 찾아줘”라고 말하면 사람은 대충 이해할 수 있지만, 컴퓨터는 “얼마 이상인지”, “어디에서 찾는지”, “결과가 여러 개면 어떻게 할지”를 알 수 없습니다.

프로그래밍 언어의 역할

프로그래밍 언어는 애매함을 줄이기 위해 엄격한 문법과 의미를 가진 약속입니다. 컴퓨터는 눈치가 없지만, 정확하게 말하면 지치지 않고 실행합니다.

프로그래밍은 절차를 설명하는 일

라면 끓이기를 컴퓨터에게 시킨다면 “적당히”라는 표현을 더 구체적인 단계로 바꾸어야 합니다.

냄비에 물 550ml를 넣는다.
물이 끓을 때까지 기다린다.
면을 넣는다.
스프를 넣는다.
4분 동안 끓인다.
불을 끈다.

4장. 왜 프로그래밍 언어는 이렇게 많을까?

모든 상황에 완벽한 언어는 없습니다. 개발하는 대상과 목적이 다르기 때문에 언어도 다양해졌습니다. 어떤 언어는 속도가 중요하고, 어떤 언어는 안전성이 중요하고, 어떤 언어는 빠른 개발과 쉬운 문법이 중요합니다.

언어 주요 특징 많이 쓰이는 분야
C 하드웨어에 가깝고 빠르지만 메모리 관리가 어렵습니다. 운영체제, 임베디드, 시스템 프로그래밍
C++ C의 성능에 객체지향과 추상화 기능을 더했습니다. 게임 엔진, 그래픽스, 고성능 시스템
C# 도구 지원이 좋고 Microsoft 생태계와 잘 맞습니다. Windows 앱, 기업 시스템, Unity
Java 기업용 서버 개발에서 널리 쓰이며 생태계가 큽니다. Spring 서버, 공공, 금융, SI
JavaScript 웹 브라우저에서 실행되는 대표 언어입니다. 웹 프론트엔드, Node.js 서버
TypeScript JavaScript에 타입 시스템을 더해 협업에 유리합니다. 대규모 웹 프론트엔드
Python 문법이 간결하고 생산성이 좋습니다. 입문, 자동화, 데이터 분석, AI, 웹

초보자에게 중요한 결론

어떤 언어가 최고인지보다, 하나의 언어로 변수, 조건문, 반복문, 함수, 자료구조, 객체, 예외 처리, 파일 처리, 데이터베이스 연결의 흐름을 익히는 것이 먼저입니다.

6장. 정보처리기사는 왜 나왔을까?

개발자는 코드만 쓰는 사람이 아닙니다. 요구사항을 이해하고, 구조를 설계하고, 데이터를 저장하고, 테스트하고, 보안을 고려하고, 문서를 작성하며 협업합니다. 정보처리기사는 이런 전반적인 IT 시스템 개발과 운영 능력을 국가기술자격 형태로 정리한 자격입니다.

정보처리기사가 말하고 싶은 것

소프트웨어는 감으로 만드는 것이 아니라 요구사항 분석, 설계, 구현, 테스트, 운영이라는 절차를 통해 만들어야 합니다.

7장. 컴퓨터 기초와 개발 실무를 연결하기

변수는 메모리와 연결된다

age = 25는 메모리 어딘가에 저장된 값을 age라는 이름으로 다루는 것입니다.

조건문은 판단을 표현한다

점수가 60점 이상이면 합격, 아니면 불합격처럼 현실의 규칙을 코드로 표현합니다.

반복문은 자동화의 핵심이다

사람은 반복 작업에 지치지만 컴퓨터는 지치지 않습니다. 반복문은 자동화의 출발점입니다.

함수는 작업에 이름을 붙인다

여러 줄의 작업을 하나의 이름으로 묶어 재사용하고, 큰 프로그램을 작은 단위로 나눕니다.

객체는 관련된 데이터와 기능을 묶는 방법

회원에게는 이름, 이메일, 가입일 같은 데이터가 있고 로그인, 비밀번호 변경 같은 동작이 있습니다. 객체지향은 이런 데이터와 기능을 함께 묶어 프로그램을 구성하는 방식입니다. 

class User {

8장. 비전공자가 자주 막히는 지점

“이해했다”와 “직접 쓸 수 있다”는 다르다

강의를 들을 때는 이해한 것 같지만 혼자 코드를 쓰려고 하면 손이 멈출 수 있습니다. 이것은 자연스러운 일입니다. 프로그래밍은 지식이면서 동시에 기술이기 때문에 직접 작성하고 실행하는 시간이 필요합니다.

에러는 실패가 아니라 대화다

에러 메시지는 컴퓨터가 보내는 힌트입니다. 다음 순서로 확인해봅니다.

  1. 에러가 난 파일은 어디인가?
  2. 몇 번째 줄인가?
  3. 에러 메시지의 핵심 단어는 무엇인가?
  4. 바로 위아래 코드에 오타가 있는가?
복붙은 나쁘지 않지만, 이해 없는 복붙은 위험하다

참고한 코드를 이해하고 자신의 문제에 맞게 바꿀 수 있어야 합니다. 입력, 출력, 바꿔야 할 부분, 에러가 날 수 있는 부분을 설명할 수 있어야 합니다.
