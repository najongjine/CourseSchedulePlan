## 이 교재의 목표

이 교재는 프로그래밍을 처음 배우는 사람이 “왜 이런 것을 배우는지”부터 이해하도록 돕기 위해 작성되었다.

프로그래밍을 처음 배우면 문법부터 마주친다. 변수, 조건문, 반복문, 함수, 클래스 같은 단어가 쏟아진다. 그런데 문법만 외우면 금방 막힌다. 왜냐하면 프로그래밍은 단순히 코드를 따라 치는 기술이 아니라, 컴퓨터에게 일을 시키기 위해 문제를 정리하고 절차로 바꾸는 사고방식이기 때문이다.

이 교재에서는 다음 질문에 답한다.

- 컴퓨터는 왜 등장했는가?
- 폰 노이만 구조는 왜 중요하며, 왜 아직도 쓰이는가?
- 프로그래밍 언어는 왜 필요한가?
- 왜 이렇게 많은 프로그래밍 언어가 생겼는가?
- C, C++, C#, Java, TypeScript, Python 같은 언어는 각각 어떤 역할을 하는가?
- 정보처리기사는 왜 생겼고, 무엇을 말하려는 자격인가?
- 정보처리기사에서 배운 내용은 실제 개발에서 어떻게 쓰이는가?

이 교재의 목적은 외우게 만드는 것이 아니라, 이후에 배울 문법과 프로젝트 수업이 어디에 연결되는지 보여주는 것이다.

# 1장. 컴퓨터는 왜 만들어졌을까?

## 1.1 컴퓨터는 원래 “계산하는 사람”이었다

오늘날 컴퓨터라고 하면 노트북, 데스크톱, 스마트폰, 서버를 떠올린다. 하지만 원래 computer라는 말은 기계가 아니라 “계산하는 사람”을 뜻했다.

과거에는 천문 계산, 항해 계산, 세금 계산, 군사 계산, 통계 계산을 사람이 직접 했다. 문제는 계산량이 너무 많고, 사람이 하다 보니 실수가 자주 발생한다는 점이었다.

예를 들어 포탄을 멀리 쏘려면 다음과 같은 정보를 계산해야 한다.

- 포탄의 무게
- 발사 각도
- 바람
- 거리
- 중력
- 지형

이 계산을 사람이 매번 손으로 하면 너무 느리다. 전쟁, 산업, 과학이 발전하면서 “빠르고 정확하게 계산하는 기계”가 필요해졌다.

컴퓨터는 처음부터 유튜브를 보고 게임을 하기 위해 만들어진 것이 아니다. 출발점은 계산이었다.

## 1.2 계산기와 컴퓨터의 차이

계산기는 정해진 계산을 빠르게 해준다.

예를 들어 계산기에 다음처럼 입력한다.

```text
3 + 5 =
```

계산기는 8을 출력한다.

하지만 컴퓨터는 단순한 계산기보다 더 넓은 일을 한다.

```text
사용자가 로그인했는가?
로그인했다면 메인 화면을 보여준다.
로그인하지 않았다면 로그인 화면으로 보낸다.
장바구니에 상품이 있는가?
있다면 결제 금액을 계산한다.
```

컴퓨터는 숫자 계산뿐 아니라 조건 판단, 반복 작업, 데이터 저장, 통신, 화면 출력까지 수행한다.

즉 컴퓨터는 “명령을 저장해두고 순서대로 실행할 수 있는 범용 기계”다.

## 1.3 컴퓨터의 핵심 능력

컴퓨터가 잘하는 일은 크게 네 가지다.

1. 빠르게 계산하기
2. 정확하게 반복하기
3. 많은 데이터를 저장하기
4. 정해진 규칙대로 판단하기

사람은 창의적이고 상황을 넓게 이해하는 데 강하다. 반면 컴퓨터는 지치지 않고 같은 일을 반복하는 데 강하다.

프로그래밍은 이 차이를 이해하는 데서 시작한다.

사람이 할 일을 컴퓨터가 할 수 있는 절차로 바꾸는 것이 프로그래밍이다.

# 2장. 폰 노이만 모델은 왜 등장했을까?

## 2.1 초기 컴퓨터의 문제

초기 컴퓨터는 지금처럼 프로그램을 쉽게 바꿀 수 없었다. 어떤 계산을 하려면 기계의 배선을 바꾸거나 스위치를 다시 설정해야 했다.

비유하면 다음과 같다.

```text
초기 컴퓨터:
요리를 바꿀 때마다 주방 구조를 뜯어고쳐야 하는 식당

현대 컴퓨터:
레시피만 바꾸면 다른 요리를 만들 수 있는 주방
```

초기 방식은 너무 불편했다. 같은 기계로 여러 작업을 하려면 “작업 내용”을 쉽게 바꿀 수 있어야 했다.

이 문제를 해결한 핵심 아이디어가 저장 프로그램 방식이다.

## 2.2 저장 프로그램 방식

저장 프로그램 방식은 프로그램도 데이터처럼 메모리에 저장하자는 생각이다.

컴퓨터가 처리하는 것은 데이터뿐 아니라 명령어도 포함된다.

```text
메모리
├─ 데이터: 10, 20, "홍길동", true
└─ 명령어: 더하라, 비교하라, 이동하라, 출력하라
```

이 아이디어 덕분에 컴퓨터는 기계 자체를 뜯어고치지 않고도 프로그램만 바꿔 여러 일을 할 수 있게 되었다.

## 2.3 폰 노이만 구조의 기본 구성

폰 노이만 구조는 현대 컴퓨터의 기본 구조를 설명하는 대표적인 모델이다.

기본 구성은 다음과 같다.

```text
입력 장치
   ↓
메모리 ↔ CPU ↔ 출력 장치
   ↑
보조 저장 장치
```

주요 구성 요소는 다음과 같다.

| 구성 요소 | 역할 |
|---|---|
| CPU | 명령어를 해석하고 실행한다 |
| 메모리 | 실행 중인 프로그램과 데이터를 임시 저장한다 |
| 입력 장치 | 키보드, 마우스, 터치처럼 외부 정보를 받는다 |
| 출력 장치 | 모니터, 스피커, 프린터처럼 결과를 보여준다 |
| 보조 저장 장치 | SSD, HDD처럼 데이터를 오래 보관한다 |

CPU는 컴퓨터의 두뇌처럼 설명되지만, 정확히는 “명령어 실행 장치”에 가깝다. CPU는 스스로 목적을 만들지 않는다. 메모리에 있는 명령을 순서대로 가져와 실행한다.

## 2.4 CPU는 어떻게 일할까?

CPU의 기본 동작은 보통 다음 세 단계로 설명한다.

```text
1. Fetch: 메모리에서 명령어를 가져온다.
2. Decode: 명령어가 무슨 뜻인지 해석한다.
3. Execute: 명령어를 실행한다.
```

예를 들어 다음과 같은 코드가 있다고 하자.

```python
total = 10 + 20
print(total)
```

컴퓨터 내부에서는 대략 다음과 같은 일이 벌어진다.

```text
10이라는 값을 준비한다.
20이라는 값을 준비한다.
두 값을 더한다.
결과 30을 total이라는 위치에 저장한다.
total의 값을 화면 출력 기능으로 보낸다.
```

우리가 보기에는 한 줄 코드지만, 컴퓨터 입장에서는 매우 작은 명령들이 순서대로 실행된다.

## 2.5 왜 아직도 폰 노이만 구조를 쓸까?

폰 노이만 구조는 오래된 모델이다. 그런데 오늘날에도 여전히 중요하다. 이유는 간단하다. 여전히 쓸모가 크기 때문이다.

첫째, 범용성이 좋다. 같은 하드웨어에서 프로그램만 바꾸면 문서 작성, 게임, 웹 브라우저, 서버, 인공지능 작업까지 수행할 수 있다.

둘째, 구조가 단순하고 이해하기 쉽다. CPU, 메모리, 입출력이라는 틀은 컴퓨터를 설명하는 데 여전히 강력하다.

셋째, 산업 전체가 이 구조를 기반으로 발전했다. 운영체제, 컴파일러, 프로그래밍 언어, 하드웨어 설계, 네트워크 시스템이 모두 이 구조를 전제로 발전했다.

물론 한계도 있다.

## 2.6 폰 노이만 병목

CPU와 메모리는 서로 데이터를 주고받아야 한다. CPU가 아무리 빨라도 메모리에서 데이터를 가져오는 속도가 느리면 전체 성능이 떨어진다. 이것을 폰 노이만 병목이라고 한다.

비유하면 다음과 같다.

```text
CPU: 요리사
메모리: 재료 창고

요리사가 아무리 빨라도 재료 창고에서 재료가 늦게 오면 요리가 늦어진다.
```

현대 컴퓨터는 이 문제를 줄이기 위해 캐시 메모리, 병렬 처리, GPU, 멀티코어, 분산 처리 같은 기술을 사용한다.

하지만 기본 아이디어, 즉 “메모리에 저장된 명령을 CPU가 순서대로 실행한다”는 생각은 여전히 프로그래밍의 중심에 있다.

# 3장. 프로그래밍 언어는 왜 필요할까?

## 3.1 컴퓨터가 직접 이해하는 언어

컴퓨터는 최종적으로 0과 1로 표현되는 기계어를 실행한다.

예를 들어 컴퓨터 내부에서는 명령과 데이터가 다음처럼 표현된다.

```text
10110000 01100001
```

사람이 이진수만 보고 프로그램을 작성하는 것은 매우 어렵다. 그래서 사람은 컴퓨터가 이해할 수 있는 명령을 더 읽기 쉬운 형태로 표현하고 싶어 했다.

이 필요에서 프로그래밍 언어가 등장했다.

## 3.2 프로그래밍 언어는 번역 가능한 약속이다

프로그래밍 언어는 사람과 컴퓨터 사이의 약속이다.

사람은 다음처럼 쓴다.

```python
print("Hello")
```

컴퓨터는 이것을 그대로 이해하지 못한다. 그래서 중간에 번역 과정이 필요하다.

```text
사람이 읽는 코드
   ↓
컴파일러 또는 인터프리터
   ↓
컴퓨터가 실행할 수 있는 명령
```

프로그래밍 언어는 사람이 문제를 표현하기 쉽게 만들고, 번역기는 그것을 컴퓨터가 실행할 수 있는 형태로 바꾼다.

## 3.3 왜 자연어로 명령하지 않을까?

사람의 말은 편하지만 애매하다.

예를 들어 “적당히 큰 숫자를 찾아줘”라는 문장은 사람에게는 어느 정도 통할 수 있다. 하지만 컴퓨터에게는 애매하다.

```text
적당히 크다 = 얼마 이상인가?
찾는다 = 어디에서 찾는가?
숫자 = 정수인가, 실수인가?
결과가 여러 개면 어떻게 할 것인가?
없으면 어떻게 할 것인가?
```

프로그래밍 언어는 애매함을 줄이기 위해 엄격한 문법과 의미를 가진다.

컴퓨터는 눈치가 없다. 대신 정확하다. 그래서 프로그래밍에서는 “정확히 말하기”가 중요하다.

## 3.4 프로그래밍은 컴퓨터에게 절차를 설명하는 일이다

라면 끓이는 법을 사람에게 설명하면 이렇게 말할 수 있다.

```text
물 끓이고, 면 넣고, 스프 넣고, 적당히 끓여.
```

하지만 컴퓨터에게 설명하려면 더 정확해야 한다.

```text
냄비에 물 550ml를 넣는다.
가스레인지를 켠다.
물이 100도에 가까워질 때까지 기다린다.
면을 넣는다.
스프를 넣는다.
4분 동안 끓인다.
불을 끈다.
그릇에 옮긴다.
```

프로그래밍도 이와 같다. 큰 문제를 작은 단계로 나누고, 컴퓨터가 실행할 수 있는 순서로 정리한다.

# 4장. 왜 프로그래밍 언어는 이렇게 많을까?

## 4.1 모든 상황에 완벽한 언어는 없다

프로그래밍 언어가 많은 이유는 개발하는 대상과 목적이 다르기 때문이다.

자동차를 고칠 때 쓰는 도구와 요리를 할 때 쓰는 도구가 다르듯, 프로그램을 만들 때도 상황에 따라 적합한 언어가 다르다.

어떤 언어는 빠른 실행 속도가 중요하다. 어떤 언어는 안전성이 중요하다. 어떤 언어는 생산성이 중요하다. 어떤 언어는 웹 브라우저에서 실행되어야 한다.

## 4.2 언어를 나누는 기준

프로그래밍 언어는 여러 기준으로 나눌 수 있다.

| 기준 | 설명 | 예 |
|---|---|---|
| 저수준/고수준 | 하드웨어에 가까운가, 사람 표현에 가까운가 | C, Python |
| 컴파일/인터프리트 | 미리 번역하는가, 실행하면서 해석하는가 | C, JavaScript |
| 정적 타입/동적 타입 | 변수 타입을 미리 엄격히 검사하는가 | Java, Python |
| 절차지향/객체지향/함수형 | 프로그램을 어떤 사고방식으로 구성하는가 | C, Java, Haskell |
| 실행 환경 | 어디에서 주로 실행되는가 | 브라우저, 서버, 모바일, 임베디드 |

초보자는 이 분류를 모두 외울 필요는 없다. 중요한 것은 언어마다 탄생 배경과 잘하는 일이 다르다는 점이다.

## 4.3 주요 언어 살펴보기

### C

C는 시스템 프로그래밍의 대표 언어다. 운영체제, 임베디드, 하드웨어에 가까운 프로그램에서 많이 사용된다.

장점은 빠르고 하드웨어 제어가 강하다는 것이다. 단점은 메모리 관리를 개발자가 직접 신경 써야 해서 어렵고 위험할 수 있다는 것이다.

```c
#include <stdio.h>

int main() {
    printf("Hello\n");
    return 0;
}
```

C를 배우면 컴퓨터가 메모리와 데이터를 어떻게 다루는지 이해하는 데 도움이 된다.

### C++

C++는 C의 성능을 유지하면서 객체지향과 다양한 추상화 기능을 추가한 언어다.

게임 엔진, 고성능 프로그램, 그래픽스, 금융 시스템, 실시간 처리 분야에서 많이 쓰인다.

```cpp
#include <iostream>

int main() {
    std::cout << "Hello" << std::endl;
    return 0;
}
```

C++는 강력하지만 학습 난도가 높은 편이다.

### C#

C#은 Microsoft 생태계에서 많이 쓰이는 언어다. Windows 프로그램, 웹 서버, 기업용 시스템, Unity 게임 개발에 사용된다.

```csharp
Console.WriteLine("Hello");
```

C#은 문법이 비교적 정돈되어 있고, 도구 지원이 좋다. Java와 비슷한 부분도 많다.

### Java

Java는 기업용 시스템과 서버 개발에서 매우 널리 쓰이는 언어다. “한 번 작성하면 여러 환경에서 실행한다”는 방향으로 발전했다.

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```

한국의 SI, 공공기관, 금융권, 대기업 서버 개발에서 Java와 Spring은 매우 중요한 조합이다.

### JavaScript와 TypeScript

JavaScript는 웹 브라우저에서 실행되는 대표 언어다.

```javascript
console.log("Hello");
```

웹 페이지에서 버튼을 눌렀을 때 반응하게 만들고, 화면을 동적으로 바꾸고, 서버와 데이터를 주고받는 데 사용된다.

TypeScript는 JavaScript에 타입 시스템을 더한 언어다.

```typescript
let name: string = "Kim";
console.log(name);
```

규모가 큰 웹 프론트엔드 프로젝트에서는 TypeScript가 많이 사용된다. 오류를 미리 잡고 협업을 쉽게 하기 위해서다.

### Python

Python은 문법이 간결하고 읽기 쉬워 입문, 자동화, 데이터 분석, 인공지능, 웹 서버 등 다양한 분야에서 사용된다.

```python
print("Hello")
```

Python의 장점은 빠르게 만들 수 있다는 것이다. 단점은 실행 속도나 대규모 시스템 구조에서 신경 써야 할 부분이 있다는 것이다.

## 4.4 우리가 주력으로 배우는 언어는 무엇인가?

교육 과정의 목표에 따라 다르지만, 국비 IT 프로그래밍 과정에서는 보통 다음 언어들이 중요하다.

| 분야 | 주력 언어 |
|---|---|
| 컴퓨터 기초 이해 | C |
| 객체지향과 서버 개발 | Java, C# |
| 웹 프론트엔드 | JavaScript, TypeScript |
| 데이터 분석과 자동화 | Python |
| 고성능 시스템과 게임 | C++ |

취업 중심의 웹 개발 과정이라면 Java, JavaScript, TypeScript, SQL이 특히 중요하다. Python은 자동화와 데이터 처리에 강하고, C는 컴퓨터 구조를 이해하는 데 도움이 된다.

초보자에게 중요한 것은 “어떤 언어가 최고인가”가 아니다. 먼저 하나의 언어로 변수, 조건문, 반복문, 함수, 자료구조, 객체, 예외 처리, 파일 처리, 데이터베이스 연결의 흐름을 익히는 것이 중요하다.

언어는 다르지만 핵심 사고는 반복된다.

# 5장. 프로그래밍 언어가 실제로 해결하는 문제

## 5.1 사람이 반복하던 일을 자동화한다

예를 들어 엑셀 파일 100개에서 매출 합계를 구해야 한다고 하자. 사람이 하나씩 열어서 계산하면 시간이 오래 걸린다.

Python으로는 이런 작업을 자동화할 수 있다.

```python
sales = [12000, 35000, 22000]
total = sum(sales)
print(total)
```

처음에는 단순해 보이지만, 이 원리는 큰 업무 자동화로 이어진다.

## 5.2 사용자의 요청을 처리한다

웹 서비스는 사용자의 요청을 받고 응답을 돌려준다.

```text
사용자: 로그인 버튼 클릭
서버: 아이디와 비밀번호 확인
서버: 성공하면 메인 화면 정보 전달
브라우저: 화면 표시
```

서버 개발자는 이 흐름을 코드로 만든다.

## 5.3 데이터를 저장하고 검색한다

프로그램은 대부분 데이터를 다룬다.

- 회원 정보
- 게시글
- 댓글
- 상품
- 주문
- 결제 내역
- 학습 기록

이 데이터는 데이터베이스에 저장된다. 그래서 개발자는 프로그래밍 언어뿐 아니라 SQL도 배운다.

```sql
SELECT name, email
FROM users
WHERE active = true;
```

프로그래밍은 데이터베이스와 함께 쓰일 때 실제 서비스가 된다.

## 5.4 현실의 규칙을 코드로 바꾼다

예를 들어 쇼핑몰 쿠폰 규칙이 있다고 하자.

```text
주문 금액이 50,000원 이상이면 5,000원 할인
단, 이미 할인 상품이면 쿠폰 적용 불가
```

코드로 표현하면 다음과 같다.

```python
price = 60000
is_discounted_item = False

if price >= 50000 and not is_discounted_item:
    price = price - 5000

print(price)
```

개발자는 현실의 정책, 규칙, 예외 상황을 코드로 바꾸는 사람이다.

# 6장. 정보처리기사는 왜 나왔을까?

## 6.1 개발자는 코드만 쓰는 사람이 아니다

처음에는 개발자를 “코드 치는 사람”이라고 생각하기 쉽다. 하지만 실제 현장에서는 코드 작성보다 더 넓은 능력이 필요하다.

개발자는 다음과 같은 일을 한다.

- 요구사항을 이해한다.
- 시스템 구조를 설계한다.
- 데이터를 어떻게 저장할지 결정한다.
- 프로그램을 구현한다.
- 테스트한다.
- 보안을 고려한다.
- 운영 중인 문제를 분석한다.
- 문서를 작성하고 협업한다.

정보처리기사는 이런 전반적인 IT 시스템 개발과 운영 능력을 국가기술자격 형태로 정리한 자격이다.

## 6.2 정보처리기사가 말하고 싶은 것

정보처리기사는 단순히 특정 언어 문법을 시험하는 자격이 아니다.

핵심 메시지는 다음과 같다.

```text
소프트웨어는 감으로 만드는 것이 아니라,
요구사항 분석, 설계, 구현, 테스트, 운영이라는 절차를 통해 만들어야 한다.
```

즉 정보처리기사는 개발의 전체 흐름을 보게 만든다.

## 6.3 정보처리기사의 주요 주제

정보처리기사에서 다루는 내용은 대략 다음과 같다.

| 주제 | 현장에서의 의미 |
|---|---|
| 요구사항 확인 | 무엇을 만들어야 하는지 정리한다 |
| 화면 설계 | 사용자가 볼 화면 흐름을 설계한다 |
| 데이터 입출력 구현 | 데이터 구조와 DB 연동을 이해한다 |
| 통합 구현 | 여러 시스템을 연결한다 |
| 서버 프로그램 구현 | 실제 비즈니스 로직을 작성한다 |
| 인터페이스 구현 | 시스템 간 약속된 방식으로 데이터를 주고받는다 |
| 애플리케이션 테스트 | 오류를 찾고 품질을 높인다 |
| SQL 응용 | 데이터베이스를 조회하고 조작한다 |
| 소프트웨어 개발 보안 | 취약점을 줄인다 |
| 프로그래밍 언어 활용 | 코드의 기본 동작 원리를 이해한다 |
| 응용 SW 기초 기술 | 운영체제, 네트워크, 데이터베이스 기초를 이해한다 |
| 제품 소프트웨어 패키징 | 배포와 문서화를 이해한다 |

이 과목들은 따로 떨어져 있는 지식처럼 보이지만, 실제로는 하나의 프로젝트 흐름에 연결된다.

## 6.4 정보처리기사에서 배운 것으로 실제로 무엇을 할 수 있을까?

정보처리기사 공부를 했다고 바로 혼자 대형 서비스를 만들 수 있는 것은 아니다. 하지만 실무를 이해하는 뼈대를 얻을 수 있다.

예를 들어 게시판을 만든다고 하자.

정보처리기사 지식은 다음처럼 연결된다.

```text
요구사항 확인:
사용자는 글을 작성, 조회, 수정, 삭제할 수 있어야 한다.

화면 설계:
목록 화면, 상세 화면, 작성 화면, 수정 화면이 필요하다.

데이터베이스:
게시글 테이블에는 제목, 내용, 작성자, 작성일이 필요하다.

서버 프로그램:
글 작성 요청이 오면 DB에 저장한다.

인터페이스:
프론트엔드와 백엔드는 JSON 형식으로 데이터를 주고받는다.

테스트:
빈 제목으로 글을 작성하면 막아야 한다.

보안:
다른 사용자의 글을 마음대로 수정하지 못하게 해야 한다.

패키징:
완성된 프로그램을 서버에 배포한다.
```

즉 정보처리기사는 “개발 프로젝트를 전체적으로 보는 지도”에 가깝다.

# 7장. 컴퓨터 기초와 개발 실무를 연결하기

## 7.1 변수는 메모리와 연결된다

프로그래밍에서 변수는 값을 담는 이름이다.

```python
age = 25
```

이 코드를 배울 때 단순히 “age에 25를 넣는다”고 외울 수 있다. 하지만 컴퓨터 구조와 연결하면 더 잘 이해된다.

```text
메모리 어딘가에 25라는 값이 저장된다.
개발자는 그 위치를 age라는 이름으로 사용한다.
```

변수는 메모리의 복잡한 주소를 사람이 이해하기 쉬운 이름으로 다루게 해준다.

## 7.2 조건문은 판단을 표현한다

```python
score = 80

if score >= 60:
    print("합격")
else:
    print("불합격")
```

조건문은 현실의 규칙을 코드로 표현한다.

```text
점수가 60점 이상인가?
맞으면 합격
아니면 불합격
```

컴퓨터는 스스로 판단하지 않는다. 개발자가 판단 기준을 코드로 써야 한다.

## 7.3 반복문은 자동화의 핵심이다

```python
for i in range(5):
    print("안녕하세요")
```

반복문은 같은 일을 여러 번 하게 만든다.

사람은 반복 작업에 지치지만 컴퓨터는 지치지 않는다. 그래서 반복문은 자동화의 핵심이다.

## 7.4 함수는 작업에 이름을 붙이는 방법이다

```python
def calculate_discount(price):
    return price * 0.9

print(calculate_discount(10000))
```

함수는 여러 줄의 작업을 하나의 이름으로 묶는다.

함수를 쓰면 다음 장점이 있다.

- 같은 코드를 반복해서 쓰지 않아도 된다.
- 코드의 의미가 분명해진다.
- 테스트하기 쉬워진다.
- 큰 프로그램을 작은 단위로 나눌 수 있다.

## 7.5 객체는 관련된 데이터와 기능을 묶는 방법이다

현실의 “회원”을 생각해보자.

회원에게는 데이터가 있다.

- 이름
- 이메일
- 비밀번호
- 가입일

회원이 할 수 있는 동작도 있다.

- 로그인한다.
- 비밀번호를 바꾼다.
- 주문한다.

객체지향 프로그래밍은 이런 데이터와 기능을 묶어서 프로그램을 구성하는 방식이다.

```java
class User {
    String name;
    String email;

    void login() {
        System.out.println(name + " 로그인");
    }
}
```

객체지향은 Java, C#, C++ 같은 언어에서 특히 중요하다.

# 8장. 비전공자가 프로그래밍을 배울 때 자주 막히는 지점

## 8.1 “이해했다”와 “직접 쓸 수 있다”는 다르다

강의를 들을 때는 이해한 것 같다. 하지만 혼자 코드를 쓰려고 하면 손이 멈춘다.

이것은 자연스러운 일이다. 프로그래밍은 지식이면서 동시에 기술이다. 수영 영상을 본다고 바로 수영을 잘할 수 없는 것처럼, 코딩도 직접 써봐야 익숙해진다.

## 8.2 에러는 실패가 아니라 대화다

초보자는 에러 메시지를 보면 당황한다. 하지만 에러는 컴퓨터가 보내는 힌트다.

예를 들어 Python에서 다음 코드는 에러가 난다.

```python
print("Hello"
```

괄호를 닫지 않았기 때문이다.

에러를 볼 때는 다음 순서로 확인한다.

```text
1. 에러가 난 파일은 어디인가?
2. 몇 번째 줄인가?
3. 에러 메시지의 핵심 단어는 무엇인가?
4. 바로 위아래 코드에 오타가 있는가?
```

에러를 무서워하지 말고 읽는 습관을 들여야 한다.

## 8.3 복붙은 나쁘지 않지만, 이해 없는 복붙은 위험하다

개발자는 실제로도 코드를 검색하고 참고한다. 중요한 것은 복사한 코드를 이해하고 자신의 문제에 맞게 바꾸는 능력이다.

다음 질문에 답할 수 있어야 한다.

- 이 코드는 어떤 입력을 받는가?
- 어떤 결과를 내는가?
- 어느 부분을 바꾸면 동작이 달라지는가?
- 에러가 나면 어디를 확인해야 하는가?

## 8.4 문법보다 중요한 것은 문제를 나누는 능력이다

“계산기 만들기”라는 과제가 있다고 하자.

처음부터 코드를 쓰려고 하면 어렵다. 먼저 문제를 나누어야 한다.

```text
1. 사용자에게 첫 번째 숫자를 입력받는다.
2. 사용자에게 연산자를 입력받는다.
3. 사용자에게 두 번째 숫자를 입력받는다.
4. 연산자가 +이면 더한다.
5. 연산자가 -이면 뺀다.
6. 연산자가 *이면 곱한다.
7. 연산자가 /이면 나눈다.
8. 결과를 출력한다.
```

이렇게 나눈 뒤에 코드를 작성하면 훨씬 쉽다.

## 과제 2. 폰 노이만 구조 그리기

# 10장. 핵심 정리

## 10.1 컴퓨터의 핵심

컴퓨터는 명령을 저장하고 실행하는 범용 기계다. 단순 계산뿐 아니라 조건 판단, 반복 처리, 데이터 저장, 통신, 화면 출력까지 수행한다.

## 10.2 폰 노이만 구조의 핵심

폰 노이만 구조의 핵심은 프로그램과 데이터를 메모리에 저장하고, CPU가 명령어를 가져와 실행한다는 것이다.

이 구조는 오래되었지만 여전히 현대 컴퓨터를 이해하는 기본 틀이다.

## 10.3 프로그래밍 언어의 핵심

프로그래밍 언어는 사람이 컴퓨터에게 일을 시키기 위해 사용하는 정확한 약속이다.

사람이 읽기 쉬운 코드를 작성하면 컴파일러나 인터프리터가 컴퓨터가 실행할 수 있는 형태로 바꾼다.

## 10.4 언어가 많은 이유

프로그래밍 언어가 많은 이유는 목적이 다르기 때문이다. 빠른 실행, 쉬운 문법, 웹 실행, 안정성, 대규모 협업, 하드웨어 제어 등 요구가 다르기 때문에 언어도 다양해졌다.

## 10.5 정보처리기사의 핵심

정보처리기사는 개발을 단순 코딩이 아니라 요구사항 분석, 설계, 구현, 테스트, 보안, 운영, 배포까지 포함하는 전체 과정으로 보게 만든다.

정보처리기사에서 배우는 내용은 실제 프로젝트의 지도 역할을 한다.

# 마무리

프로그래밍을 처음 배울 때 가장 중요한 것은 모든 것을 한 번에 이해하려 하지 않는 것이다.

처음에는 컴퓨터가 어떻게 생각하는지 익숙하지 않다. 하지만 변수는 메모리와 연결되고, 조건문은 판단과 연결되며, 반복문은 자동화와 연결된다. 함수는 복잡한 일을 나누는 방법이고, 객체는 현실의 개념을 코드로 정리하는 방법이다.

정보처리기사는 이 모든 것을 더 큰 개발 프로젝트의 흐름 속에서 보게 해준다.

결국 프로그래밍은 컴퓨터 언어를 배우는 일이면서 동시에 문제를 정리하는 법을 배우는 일이다.
